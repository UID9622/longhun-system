# MCP身份认证集成指南

## 🔐 什么是MCP？

MCP (Model Context Protocol) 身份认证系统是CNSH安全屋的核心安全组件，提供以下功能：

- **二维码认证** - 扫码授权后才能访问AI功能
- **设备管理** - 管理已授权的设备，支持7天免认证
- **安全日志** - 记录所有认证尝试和授权操作
- **本地化存储** - 所有数据仅存储在本地，保护隐私

## 🚀 快速开始

### 1. 初始化MCP系统

在首次使用前，需要初始化MCP认证系统：

5. 扫描二维码或手动访问认证链接
6. 输入密码（默认：CNSH2025）
7. 完成认证

## 📱 认证流程

1. **请求认证** - MulanNotion请求认证令牌
2. **生成二维码** - 桥接服务器生成认证二维码
3. **扫码认证** - 用户扫码访问认证页面
4. **输入密码** - 用户输入密码进行验证
5. **授权设备** - 认证成功后授权设备
6. **状态同步** - 认证状态同步回MulanNotion

## 🔧 故障排除

### 问题1：认证服务器未运行

**症状：** 提示"认证服务器未运行，请先启动服务器"

**解决方案：**
```bash
./mcp-init.sh
```

### 问题2：二维码无法显示

**症状：** 认证对话框中二维码区域空白

**解决方案：**
1. 检查网络连接
2. 确保已加载qrcode.js库
3. 查看浏览器控制台错误信息

### 问题3：认证失败

**症状：** 输入正确密码但提示认证失败

**解决方案：**
1. 检查认证服务器是否运行
2. 确认密码正确
3. 检查设备时间和系统时间是否一致

### 问题4：设备未授权

**症状：** 认证成功但状态仍显示"未认证"

**解决方案：**
1. 刷新页面
2. 检查设备ID是否正确
3. 查看浏览器控制台错误信息

## 📋 常用命令

### 查看MCP服务状态
```bash
# 检查认证服务器是否运行
curl http://localhost:5000/status

# 检查桥接服务器是否运行
curl http://localhost:[桥接端口]/status
```

### 查看已授权设备
```bash
curl http://localhost:5000/devices
```

### 撤销设备授权
```bash
# 替换DEVICE_ID为要撤销的设备ID
curl -X DELETE http://localhost:5000/revoke/DEVICE_ID
```

### 停止所有MCP服务
```bash
./mcp-stop.sh
```

## 🔒 安全注意事项

1. **定期更换密码** - 建议每月更换一次认证密码
2. **检查设备列表** - 定期检查已授权设备列表，撤销不认识的设备

4. **备份认证数据** - 定期备份认证日志和设备列表
