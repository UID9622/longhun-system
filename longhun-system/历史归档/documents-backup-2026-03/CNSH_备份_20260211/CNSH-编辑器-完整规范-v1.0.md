## 一、语言核心规范

### 1.1 设计原则

```
中文编程：关键字/结构/语义以中文为一等公民
DNA 追溯：每个文件/函数/规则可绑定 DNA 追溯码
三色审计：编译前强制审计（红阻断、黄提示、绿放行）
算力节约：能用模板解决的不推理，能复用的不重复计算
```

### 1.2 词法规范

**注释：**

```
# 单行注释：从 # 到行尾
```

**标识符：**

```
允许：中文、英文字母、数字、下划线
规则：首字符不能是数字

示例：
文本 名称
整数 用户_编号1
```

**字面量：**

```
整数：0 123
小数：3.14
文本："..." '...' 「...」 『...』
真假：真 假
空：空
```

**关键字：**

```
类型：整数 小数 文本 真假 空值
控制：如果 否则 循环 返回
结构：函数 返回类型
输入输出：打印 输入
常量：真 假 空
元信息：DNA追溯 三色审计
```

### 1.3 语法规范（BNF）

```bnf
<程序> ::= { <语句> } EOF

<语句> ::= <变量声明>
        |  <函数声明>
        |  <如果语句>
        |  <循环语句>
        |  <返回语句>
        |  <打印语句>
        |  <表达式语句>
        |  <元信息语句>

<变量声明> ::= <类型> <标识符> [ "=" <表达式> ] [";"]

<类型> ::= "整数" | "小数" | "文本" | "真假" | "空值"

<函数声明> ::= "函数" <标识符> "(" [ <参数列表> ] ")"
               [ "返回类型" <类型> ]
               "{" { <语句> } "}"

<如果语句> ::= "如果" <条件括号> "{" { <语句> } "}"
               [ "否则" "{" { <语句> } "}" ]

<条件括号> ::= "【" <表达式> "】" | "[" <表达式> "]"

<循环语句> ::= "循环" <条件括号> "{" { <语句> } "}"

<返回语句> ::= "返回" [ <表达式> ] [";"]

<打印语句> ::= "打印" <表达式> [";"]

<元信息语句> ::= "DNA追溯" ":" <文本字面量>
              | "三色审计" ":" <文本字面量>
```

### 1.4 标准示例

```
# DNA追溯码：#ZHUGEXIN-2026-01-27-HELLO-WORLD

    打印「你好，CNSH」
    
    整数 成本 = 80
    整数 售价 = 120
    整数 利润 = 售价 - 成本
    
    如果【利润 > 0】{
        打印「有利润」
    } 否则 {
        打印「亏损」
    }
    
    循环【3】{
        打印「测试循环」
    }
    
    返回 0

```

## 二、标准库接口

### 2.1 输出与提示（3 个）

```
函数 打印(文本 内容) 返回类型 空值
# 正常输出

函数 提示(文本 内容) 返回类型 空值
# 黄色级别提示，不阻断

函数 报错(文本 内容) 返回类型 空值
# 红色级别报错，阻断当前执行
```

### 2.2 输入与确认（2 个）

```
函数 输入文本(文本 提示语) 返回类型 文本
# 获取用户输入

函数 确认(文本 提示语) 返回类型 真假
# 是/否确认
```

### 2.3 时间与日志（3 个）

```
函数 当前时间() 返回类型 文本
# 返回当前时间

函数 记录日志(文本 级别, 文本 内容) 返回类型 空值
# 写入系统日志

函数 生成追溯(文本 标签) 返回类型 文本
# 生成 DNA 子码
```

### 2.4 字符串与格式（2 个）

```
函数 拼接(文本 A, 文本 B) 返回类型 文本
# 拼接两个文本

函数 包含(文本 全文, 文本 关键词) 返回类型 真假
# 检查是否包含关键词
```

### 2.5 安全与审计（2 个）

```
函数 三色审计(文本 内容) 返回类型 文本
# 返回：红/黄/绿 + 原因

函数 熔断(文本 原因) 返回类型 空值
# 立即停止执行
```

### 2.6 使用示例

```
函数 审计示例() 返回类型 整数 {
    文本 内容 = 「我要测试一段话」
    文本 结果 = 三色审计(内容)
    
    如果【包含(结果, 「红」)】{
        报错(拼接(「审计阻断：」, 结果))
        返回 1
    }
    
    如果【包含(结果, 「黄」)】{
        提示(拼接到「审计提醒：」, 结果))
    }
    
    打印(拼接到「审计通过：」, 结果))
    打印(内容)
    
    返回 0
}
```

## 三、DNA 追溯机制

### 3.1 DNA 格式定义

```
格式：#创作者ID-日期-项目名-版本-哈希值

示例：
#UID9622-2026-01-27-记账软件-v1.0-a1b2c3d4

生成算法：
Hash = SHA256(文件内容 + 时间戳 + 用户签名)
DNA = #UserID-Date-ProjectName-Version-Hash[:8]
```

### 3.2 DNA 嵌入方式

**在文件头部添加注释：**

```
# DNA追溯码：#UID9622-2026-01-27-记账软件-v1.0-a1b2c3d4
# 创作者：诸葛鑫（UID9622）

# GPG签名：A2D0092CEE2E5BA87035600924C3704A8CC26D5F

函数 主函数() {
    # 代码内容
}
```

## 四、三色审计系统

### 4.1 审计级别定义

```yaml
levels:
  red:
    description: "明确违法/明确伤害/明确越权/不可逆破坏"
    action: "block"
  yellow:
    description: "敏感/可能隐私泄露/高风险但不确定"
    action: "warn"
  green:
    description: "通过"
    action: "allow"
```

### 4.2 审计规则包（YAML 格式）

```yaml
version: "v1.0.0"
dna_prefix: "#ZHUGEXIN"

policy:
  mode: "strict"
  default_level: "green"
  red_action: "block"
  yellow_action: "warn"
  green_action: "allow"
  log: true
  broadcast: true

# 红色规则（阻断）
rules_red:
  - id: "AUDIT-R-001"
    name: "暴力伤害"
    match:
      any_regex:
        - "杀人|灭口|下毒|爆炸|炸弹|枪支改造|自制武器|屠杀|恐袭"
    reason: "明确伤害/暴力"
    suggestion: "仅允许输出安全与法律风险提醒"

  - id: "AUDIT-R-002"
    name: "犯罪与诈骗"
    match:
      any_regex:
        - "诈骗|洗钱|贩毒|制毒|走私|博彩漏洞|黑产|外挂售卖"
    reason: "违法与犯罪引导"
    suggestion: "仅允许输出合规教育、反诈提示"

  - id: "AUDIT-R-003"
    name: "黑客入侵与破坏"
    match:
      any_regex:
        - "入侵|提权|爆破|绕过验证|后门|免杀|木马|钓鱼链接制作|勒索"
    reason: "明确越权/入侵/破坏"
    suggestion: "仅允许输出防护、加固建议"

  - id: "AUDIT-R-004"
    name: "不可逆破坏指令"
    match:
      any_regex:
        - "删库|rm -rf|格式化硬盘|清空数据|销毁证据"
    reason: "不可逆破坏"
    suggestion: "仅允许输出数据保护/备份方案"

# 黄色规则（提示）
rules_yellow:
  - id: "AUDIT-Y-001"
    name: "疑似隐私/证件号"
    match:
      any_regex:
        - "\\b\\d{15,18}\\b"
        - "\\b\\d{16}\\b"
    reason: "可能包含证件号/银行卡号"
    suggestion: "提示用户打码"

  - id: "AUDIT-Y-002"
    name: "疑似密钥/凭证"
    match:
      any_regex:
        - "AKIA[0-9A-Z]{16}"
        - "-----BEGIN (RSA|OPENSSH|EC) PRIVATE KEY-----"
        - "sk-[A-Za-z0-9]{20,}"
    reason: "可能包含密钥或私钥"
    suggestion: "提示撤回并更换密钥"

  - id: "AUDIT-Y-003"
    name: "高争议敏感话题"
    match:
      any_regex:
        - "宗教冲突|极端主义|政治煽动|仇恨言论"
    reason: "可能引发社会风险"
    suggestion: "仅提供中立科普、合规边界"

# 绿色规则（放行）
rules_green:
  - id: "AUDIT-G-001"
    name: "通用知识/工具使用"
    match:
      any_regex:
        - "Notion|数据库|模板|整理|复盘|学习方法"
    reason: "正常请求"
    suggestion: "允许完整回答"
```

### 6.4 合规声明

**合规承诺：**

- 系统遵守用户所在地与服务提供方所在地的适用法律
- 不提供违法操作指引
- 默认以更严格标准保护用户与系统

**隐私承诺：**

- 不以追踪为目的收集个人敏感信息
- 仅在必要范围内做合规判断
- 记录仅用于审计与安全（DNA追溯）

**敏感议题处理规则：**

- 系统不进行煽动、站队、攻击性输出
- 若用户要求对敏感议题下结论：

    ◦ 仅提供中性说明 + 官方信息查询路径

    ◦ 或提示该议题存在争议/法域差异，避免误导

- 任何地图/国别/地区表述：

    ◦ 以用户所在法域/发布平台规则/官方公开表述为准

    ◦ 系统不擅自扩写或发明结论

**免责声明：**

- 本系统输出不构成法律意见
- 重大事项请咨询专业律师或权威机构

```
   - 创建者 UID
   - 创建时间

   - 审计结果

   - 红色：阻断执行
```

## 十二、对外介绍（普通人版）

### 12.1 这是什么

用中文写"能跑的代码"，让普通人也能做自动化。

你会用中文写话 → 就能写程序

你不懂编程 → 也能把事情自动跑起来

### 12.2 能帮你干什么

**办公：**

- 自动整理表格
- 批量生成文档
- 一键检查数据

**生意：**

- 统计每天流水
- 自动算账
- 生成报表

**生活：**

- 定时提醒
- 简单规则自动执行

### 12.3 为什么不一样

**中文写代码：**

- 不记符号
- 看得懂、敢去改

**自动合规：**

- 系统自己检查风险
- 用错了会拦住你

**责任清楚：**

- 谁写的、谁改的，都有记录
