## 📊 Database 1：算法总览

### 数据库属性

```yaml
标题: AI算法总览
类型: Table
视图: 
  - 全部算法（表格）
  - 按类别分组（看板）
  - 学习进度（日历）
  - 重要度排序（列表）
```

### 属性字段

```yaml
1. 名称 (Title)
   类型: 标题
   示例: "线性回归"

2. 算法类别 (Select)
   选项:
     - 传统机器学习
     - 深度学习
     - 优化算法
     - 强化学习
   颜色: 蓝/红/绿/黄

3. 重要程度 (Select)
   选项:
     - ⭐⭐⭐ 基础必学
     - ⭐⭐ 主流常用
     - ⭐ 前沿热点
   颜色: 红/黄/绿

4. 学习状态 (Status)
   选项:
     - 🔵 未开始
     - 🟡 学习中
     - 🟢 已完成
     - 🔴 需复习

5. 难度等级 (Select)
   选项:
     - 入门
     - 中级
     - 高级
     - 专家

6. 学习时间 (Number)
   格式: 周
   示例: 4（表示4周）

7. 应用领域 (Multi-select)
   选项:
     - 集成电路
     - 工业母机
     - 高端仪器
     - 基础软件
     - 先进材料
     - 新能源
     - 生物制造
     - 人工智能

8. 龍魂融合 (Select)
   选项:
     - ✅ 已融合
     - 🟡 规划中
     - ⚪ 未融合

9. 理论文档 (Files & media)
   上传: theory.md

10. 代码实现 (Files & media)
    上传: code.py

11. DNA追溯码 (Text)
    格式: #龍芯⚡️日期-算法名-版本

12. 开始日期 (Date)

13. 完成日期 (Date)

14. 学习笔记 (Relation)
    关联到: 学习路径Database

15. 应用案例 (Relation)
    关联到: 应用案例Database

16. 实验记录 (Relation)
    关联到: 实验记录Database
```

## 🔗 数据库关联关系

```yaml
算法总览 ←→ 学习路径
  → 一个算法对应4周学习路径
  → 关系: Many-to-Many

算法总览 ←→ 应用案例
  → 一个算法可用于多个案例
  → 一个案例可能用多个算法
  → 关系: Many-to-Many

算法总览 ←→ 实验记录
  → 一个算法有多次实验
  → 关系: One-to-Many

算法总览 ←→ 进度追踪
  → 多个算法属于一个季度
  → 关系: Many-to-One

学习路径 ←→ 实验记录
  → 学习过程中的实验记录
  → 关系: One-to-Many
```

### Step 2：导入初始数据

```python
# 使用Notion API批量创建页面

import requests

NOTION_TOKEN = "secret_你的Token"
DATABASE_ID = "你的数据库ID"

# 创建线性回归页面
page_data = {
    "parent": {"database_id": DATABASE_ID},
    "properties": {
        "名称": {"title": [{"text": {"content": "线性回归"}}]},
        "算法类别": {"select": {"name": "传统机器学习"}},
        "重要程度": {"select": {"name": "⭐⭐⭐ 基础必学"}},
        "学习状态": {"status": {"name": "🟢 已完成"}},
        "难度等级": {"select": {"name": "入门"}},
        "学习时间": {"number": 4},

            {"name": "集成电路"},

        ]},
        "龍魂融合": {"select": {"name": "✅ 已融合"}},
        "DNA追溯码": {"rich_text": [{"text": {"content": "#龍芯⚡️2026-01-15-线性回归-v1.0"}}]}
    }

    "https://api.notion.com/v1/pages",
```
