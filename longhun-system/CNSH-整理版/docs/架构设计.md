# 🏗️ CNSH 架构设计

> 本文档描述 CNSH 龙魂系统的整体架构设计。

## 1. 系统概述

### 1.1 设计目标

CNSH 是一个面向中文用户的**去中心化数字生态系统**，设计目标包括：

| 目标 | 说明 |
|------|------|
| **技术自主** | 完全开源，不依赖国外闭源技术 |
| **数据主权** | 用户数据完全本地化存储 |
| **中文原生** | 从底层设计支持中文语境 |
| **普惠易用** | 降低技术门槛，人人可用 |

### 1.2 核心组件

## 2. 字元编辑器架构

### 2.1 技术选型

| 层次 | 技术 | 理由 |
|------|------|------|
| 渲染引擎 | HTML5 Canvas 2D | 跨平台，无需构建 |
| 矢量输出 | SVG | 标准格式，可无限缩放 |
| 数据格式 | JSON (.cnsh) | 易读易解析 |

### 2.2 渲染管线

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 输入阶段  │───►│ 处理阶段  │───►│ 渲染阶段  │───►│ 输出阶段  │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
      │              │              │              │
      ▼              ▼              ▼              ▼
  鼠标/触摸      点集简化      15层渲染管线     SVG/Canvas
  采集坐标       曲线拟合      • 笔画力度       PNG导出
  时间戳记       贝塞尔转换    • 侵蚀效果       .cnsh保存
                               • 纹理映射
                               • 墨色渐变
```

## 3. AI核心系统架构

### 3.1 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      CNSH Core                              │
│                    (Node.js + Express)                      │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐        │
│  │   API层     │  │   服务层    │  │   数据层    │        │
│  │             │  │             │  │             │        │
│  │ • HTTP API  │  │ • 知识服务  │  │ • SQLite    │        │
│  │ • WebSocket │  │ • AI服务    │  │ • 向量DB    │        │
│  │ • 插件接口  │  │ • 同步服务  │  │ • 文件系统  │        │
│  └──────┬──────┘  └──────┬──────┘  └──────┬──────┘        │
│         │                │                │               │
│         └────────────────┴────────────────┘               │
│                          │                                │
│                          ▼                                │
│                   ┌─────────────┐                         │
│                   │  Ollama SDK │                         │
│                   │  (本地LLM)  │                         │
│                   └─────────────┘                         │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 模块职责

| 模块 | 职责 | 关键类/文件 |
|------|------|------------|
| `server.js` | HTTP服务入口 | `CNSHServer` |
| `routes/` | 路由定义 | `knowledge.js`, `ollama.js` |
| `services/` | 业务逻辑 | `embedding.js`, `dailyReview.js` |
| `utils/` | 工具函数 | `logger.js`, `vector-db.js` |

### 3.3 数据流

```
用户输入 → API路由 → 服务处理 → 向量编码 → 数据存储
                ↓
           LLM推理 ← Ollama ← 本地模型
                ↓
用户输出 ← API响应 ← 结果格式化 ← 向量检索
```

## 4. DNA 编码系统

### 4.1 编码规则

每个 CNSH 组件/作品都有唯一的 DNA 编码：

```
格式：#{前缀}⚡️-CNSH-{类型}-{序号}

示例：
- #龍芯⚡️-CNSH-EDITOR-0001    (编辑器)
- #龍芯⚡️-CNSH-CORE-0001      (核心系统)
- #龍芯⚡️-CNSH-GLYPH-0001     (字元作品)
- #龍芯⚡️-CNSH-COMMIT-0001    (代码提交)
```

### 4.2 类型定义

| 类型 | 说明 |
|------|------|
| `EDITOR` | 字元编辑器相关 |
| `CORE` | 核心系统相关 |
| `PLUGIN` | Obsidian插件相关 |
| `GLYPH` | 用户创作的汉字字元 |
| `COMMIT` | 代码提交 |
| `DOCS` | 文档 |

## 6. 扩展性设计

### 6.1 插件系统

```javascript
// 插件接口定义
interface CNSHPlugin {
  name: string;           // 插件名称
  version: string;        // 版本号
  
  initialize(): void;     // 初始化
  process(data: any): any; // 数据处理
}

// 示例插件
const 示例插件 = {
  name: "书法风格迁移",
  version: "1.0.0",
  
  initialize() {
    console.log("插件已加载");
  },
  
  process(字元数据) {
    // 应用风格迁移
    return 处理后的字元;
  }
};
```

### 6.2 API 设计

```yaml
# RESTful API 规范
/api/v1/
  /glyphs:
    GET:    获取字元列表
    POST:   创建新字元
    
  /glyphs/{id}:
    GET:    获取字元详情
    PUT:    更新字元
    DELETE: 删除字元
    
  /knowledge:
    GET:    搜索知识
    POST:   创建笔记
    
  /chat:
    POST:   AI对话
    
  /sync:
    POST:   触发同步
```
