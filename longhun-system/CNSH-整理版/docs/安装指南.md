# 📦 CNSH 安装指南

> 本指南将帮助你安装和配置 CNSH 龍魂体系。

---

## 🎯 安装方式选择

| 需求 | 推荐方式 | 所需组件 |
|------|----------|----------|
| 仅使用字元编辑器 | [方式一](#方式一仅安装字元编辑器) | 浏览器 |

## 方式一：仅安装字元编辑器

字元编辑器是**纯前端应用**，无需服务器，双击即可使用。

### 步骤

```bash
# 1. 克隆仓库（或直接下载 ZIP）
git clone https://github.com/UID9622/CNSH.git

# 2. 进入编辑器目录
cd CNSH/packages/cnsh-editor

# 3. 打开快速启动页面
open 快速启动.html        # macOS
xdg-open 快速启动.html    # Linux
start 快速启动.html       # Windows
```

或使用 Python 启动本地服务器：

```bash
cd CNSH/packages/cnsh-editor
python3 -m http.server 8080

# 浏览器访问 http://localhost:8080
```

### 浏览器兼容性

| 浏览器 | 支持状态 |
|--------|----------|
| Chrome 90+ | ✅ 完全支持 |
| Firefox 88+ | ✅ 完全支持 |
| Edge 90+ | ✅ 完全支持 |
| Safari 14+ | ✅ 支持（部分特性） |

## 方式二：完整系统安装

完整系统包含字元编辑器 + AI核心 + Obsidian插件。

### 前置要求

| 组件 | 版本 | 用途 |
|------|------|------|
| Node.js | v18+ | 后端服务 |
| Python | 3.8+ | AI处理脚本 |
| Ollama | v0.1.0+ | 本地大模型 |
| Obsidian | v1.0.0+ | 知识管理（可选） |

### 一键安装

```bash
# 1. 克隆仓库
git clone https://github.com/UID9622/CNSH.git
cd CNSH

# 2. 运行安装脚本
chmod +x scripts/install.sh
./scripts/install.sh
```

### 手动安装

#### 1. 安装 Ollama

```bash
# macOS/Linux
curl -fsSL https://ollama.ai/install.sh | sh

# 验证安装
ollama --version
```

#### 2. 下载中文模型

```bash
# 下载推荐模型
ollama pull qwen:7b-chat
ollama pull chatglm3:6b

# 查看已安装模型
ollama list
```

#### 3. 安装 CNSH 核心

```bash
cd packages/cnsh-core

# 安装 Node 依赖
npm install

# 安装 Python 依赖
pip install -r requirements.txt

# 复制环境变量模板
cp .env.example .env

# 编辑 .env 文件（根据需要修改）
nano .env
```

#### 4. 配置环境变量

编辑 `.env` 文件：

```bash
# CNSH 核心配置
CNSH_PORT=3000
CNSH_HOST=localhost
CNSH_LOG_LEVEL=info

# Ollama 配置
OLLAMA_HOST=http://localhost:11434
OLLAMA_MODEL=qwen:7b-chat
OLLAMA_TIMEOUT=30

# Obsidian 配置（可选）
OBSIDIAN_VAULT_PATH=/path/to/your/vault

# 数据库配置
DB_TYPE=sqlite
DB_PATH=./data/cnsh.db
```

#### 5. 启动服务

```bash
# 启动 CNSH 核心服务
npm start

# 或后台运行
npm run start:daemon
```

#### 6. 安装 Obsidian 插件（可选）

```bash
# 1. 打开 Obsidian
# 2. 设置 → 第三方插件 → 浏览社区插件
# 3. 搜索 "CNSH Connector"
# 4. 安装并启用
# 5. 在插件设置中配置服务器地址：http://localhost:3000
```

或者手动安装：

```bash
# 复制插件到 Obsidian 插件目录
cp -r packages/cnsh-plugin /path/to/your/vault/.obsidian/plugins/cnsh-connector
```

## 方式三：开发环境

开发者需要安装完整的开发依赖。

### 安装开发依赖

```bash
cd CNSH

# 安装所有包依赖
npm run install:all

# 安装 Git 钩子
npm run prepare
```

### 启动开发模式

```bash
# 同时启动前端和后端
npm run dev

# 单独启动编辑器（带热重载）
npm run dev:editor

# 单独启动核心服务（带自动重启）
npm run dev:core
```

## 🧪 验证安装

### 验证字元编辑器

```bash
# 启动编辑器
cd packages/cnsh-editor
python3 -m http.server 8080

# 访问 http://localhost:8080
# 尝试绘制一个笔画，应该能看到渲染效果
```

### 验证核心系统

```bash
# 1. 检查服务状态
curl http://localhost:3000/api/health

# 预期输出：
# {"status":"ok","version":"x.x.x"}

# 2. 检查 Ollama 连接
curl http://localhost:3000/api/ollama/status

# 3. 测试 AI 对话
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{"message":"你好"}'
```

## 🛠️ 故障排除

### 常见问题

#### 1. 端口被占用

```bash
# 查找占用 3000 端口的进程
lsof -i :3000

# 修改 CNSH 端口（编辑 .env）
CNSH_PORT=3001
```

#### 2. Ollama 连接失败

```bash
# 检查 Ollama 服务是否运行
ollama serve

# 测试 Ollama API
curl http://localhost:11434/api/tags
```

#### 3. Obsidian 插件无法连接

```bash
# 检查 CNSH 服务是否可访问
curl http://localhost:3000/api/health

# 检查浏览器控制台错误信息
# 确保没有 CORS 问题
```

#### 4. Python 依赖安装失败

```bash
# 使用虚拟环境
python3 -m venv venv
source venv/bin/activate  # Linux/Mac
# 或 venv\Scripts\activate  # Windows

pip install -r requirements.txt
```
