## 一、字元编辑器

### 1.1 界面介绍

```
┌─────────────────────────────────────────────────────┐
│  CNSH 字元编辑器                          [菜单]   │
├─────────────────────────────────────────────────────┤
│  字元名称: [________________]  [保存] [导出SVG]    │
├─────────────────────────────────────────────────────┤
│                                                     │
│                    ┌───────────┐                    │
│                    │           │                    │
│     ┌──────┐       │  画布     │       ┌──────┐    │
│     │ 参数 │       │           │       │ 图层 │    │
│     │ 面板 │       │  (在此    │       │ 面板 │    │
│     │      │       │   绘图)   │       │      │    │
│     └──────┘       │           │       └──────┘    │
│                    └───────────┘                    │
│                                                     │
├─────────────────────────────────────────────────────┤
│  状态: 就绪 | DNA: #龍芯⚡️-CNSH-GLYPH-0001      │
└─────────────────────────────────────────────────────┘
```

### 1.2 基础操作

#### 绘制笔画

1. 在**字元名称**输入框中输入要创作的字（如"和"）
2. 在画布上按住鼠标左键拖动
3. 松开鼠标完成笔画
4. 重复步骤 2-3 完成整个字

#### 调整参数

| 参数 | 范围 | 效果 |
|------|------|------|
| 笔画力度 | 8-30 | 笔画粗细 |
| 侵蚀强度 | 0-1.0 | 风化效果 |
| 纹理类型 | 光滑/粗糙/石刻 | 表面质感 |
| 墨色浓度 | 0.4-1.0 | 颜色深浅 |

### 1.3 高级功能

#### 保存和导出

```javascript
// 保存为 .cnsh 格式（可重新编辑）
点击 [保存] 按钮 → 下载 .cnsh 文件

// 导出为 SVG 矢量图
点击 [导出SVG] 按钮 → 下载 .svg 文件
```

#### DNA 追溯

每个创建的字元都会自动生成唯一的 DNA 编码：

```
格式：#龍芯⚡️-CNSH-GLYPH-{时间戳}

示例：#龍芯⚡️-CNSH-GLYPH-1706789123456
```

这个编码永久标识你的字元作品。

### 1.4 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl/Cmd + S` | 保存字元 |
| `Ctrl/Cmd + E` | 导出 SVG |
| `Ctrl/Cmd + Z` | 撤销（开发中） |
| `Delete` | 删除选中笔画（开发中） |

## 二、AI核心系统

### 2.1 系统架构

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│   Obsidian      │◄───►│   CNSH Core     │◄───►│    Ollama       │

└─────────────────┘     └─────────────────┘     └─────────────────┘
                               │
                               ▼
                        ┌─────────────────┐
                        │   SQLite DB     │
                        │   (向量数据库)   │
                        └─────────────────┘
```

### 2.2 知识管理

#### 创建知识笔记

```bash
# 通过 API 创建笔记
curl -X POST http://localhost:3000/api/knowledge/notes \
  -H "Content-Type: application/json" \
  -d '{
    "title": "我的第一篇笔记",
    "content": "这是笔记内容...",
    "tags": ["标签1", "标签2"]
  }'
```

#### 搜索知识

```bash
# 语义搜索
curl "http://localhost:3000/api/knowledge/search?q=人工智能"

# 标签搜索
curl "http://localhost:3000/api/knowledge/tags/标签1"
```

### 2.3 智能对话

#### 基础对话

```bash
# 发送消息
curl -X POST http://localhost:3000/api/chat \
  -H "Content-Type: application/json" \
  -d '{
    "message": "解释一下量子计算",
    "context": "previous_conversation_id"
  }'
```

#### 基于知识库的问答

```bash
# 问答模式（自动检索相关知识）
curl -X POST http://localhost:3000/api/knowledge/ask \
  -H "Content-Type: application/json" \
  -d '{
    "question": "我在笔记中提到的那个项目是什么？"
  }'
```

### 2.4 Notion 同步

#### 配置 Notion 集成

1. 获取 Notion Integration Token
2. 在 `.env` 文件中配置：

```bash
NOTION_TOKEN=secret_xxx
NOTION_DATABASE_ID=xxx
```

#### 同步命令

```bash
# 手动同步
./scripts/sync-notion.sh

# 启动自动同步（每 5 分钟）
./scripts/notion-monitor-daemon.sh start
```

## 三、Obsidian插件

### 3.1 安装

#### 方式一：社区插件

1. 打开 Obsidian → 设置 → 第三方插件
2. 关闭安全模式
3. 浏览社区插件
4. 搜索 "CNSH Connector"
5. 安装并启用

#### 方式二：手动安装

```bash
# 复制插件文件到插件目录
cp -r packages/cnsh-plugin \
  /path/to/vault/.obsidian/plugins/cnsh-connector

# 重启 Obsidian
```

### 3.2 配置

#### 插件设置

```json
{
  "cnsh_server": "http://localhost:3000",
  "api_key": "your_api_key",
  "model": "qwen:7b-chat",
  "auto_tag": true,
  "smart_notes": true,
  "knowledge_graph": true
}
```

## 四、常见问题

### Q1: 字元编辑器无法保存文件？

浏览器限制直接保存文件到本地。解决方法：

1. 点击保存按钮后，手动下载弹出的文件
2. 或使用本地服务器方式运行编辑器

### Q2: AI 回复很慢？

本地模型运行速度取决于硬件：

- **推荐配置**: 16GB+ 内存，支持 AVX2 的 CPU
- **GPU 加速**: 配置 Ollama 使用 GPU

```bash
# 检查 Ollama 是否使用 GPU
ollama ps
```

### Q3: 如何备份数据？

```bash
# 备份脚本
./scripts/backup.sh

# 手动备份
cp -r data/ backup/$(date +%Y%m%d)/
```

## 五、技巧与最佳实践

### 字元创作技巧

1. **分层绘制**: 先画轮廓，再填充细节
2. **参数搭配**: 石刻纹理 + 高侵蚀 = 古碑效果
3. **保存习惯**: 定期保存 .cnsh 源文件

### 知识管理技巧

1. **标签系统**: 使用一致的标签命名规范
2. **定期回顾**: 使用每日回顾功能
3. **链接笔记**: 多使用 Obsidian 的 `[[链接]]`
