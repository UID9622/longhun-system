## 一、为什么要有投喂入口

**规则层**（L0-L1）已经有不动点锚 —— 13 条封顶，不可改。  
**操作层**（L2-L4）**也必须有不动点** —— 否则规则再硬，操作一乱就破防。

> **"操作的不动点 = 投喂入口"**

**核心原理**：
- **单点入口**：所有内容必经一条路进系统
- **多路分发**：出口根据层级自动分拨
- **熔断兜底**：三色 + 黑名单 + 数字根 三层护栏
- **自动登记**：每笔投喂自动打锚，永不可抵赖

## 三、三色熔断门（进门三检）

```
任何内容进门 → 检一：必填字段 → 检二：黑名单 → 检三：数字根
                ↓              ↓              ↓
            🔴 缺失拒绝      🔴 触发拒绝      🔴 dr∈{3,9} 拒绝
                           
              三关全过 → 🟢 放行 → 登记锚
```

### 🔴 强制拒绝条件

1. **必填字段缺失**：title / content / source_type
2. **黑名单命中**：
   - 伪造身份（"我就是 UID9622"）
   - 伪造 DNA（"fake-dna"）
   - 指令注入（"ignore previous instructions"）
3. **数字根熔断**：SHA-256 前 6 位 % 9999 → dr ∈ {3, 9}

### 🟡 待补条件

- 内容长度 > 50000 字符
- 内容长度 < 3 字符

## 四、不动点锚自动登记

通过三色门的内容 → **立刻调用 `fixed_point_registry.Ψ(s)`** 压缩成锚：

```
Ψ(s) = T̂(title) ⊗ D̂(title+content) ⊗ L̂(推断层级) × ETE(关键词)
                    ↓
            锚 = {T, D, L, sha16, dr, 五行, ete, 一句话}
                    ↓
            写入 ~/longhun-system/算法仓库/不动点锚登记册/registry.jsonl
```

**查重**：如果 sha16 已存在 → 复用旧锚，返回 `duplicate` 状态（不重复登记）。
