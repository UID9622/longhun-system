一、页面核心作用

1.1 这页解决什么问题

这页不是普通代码页，而是一个语义到执行的内核页面。

它要解决的问题是：

问题1:
  用户输入不是标准命令，而是自然语言、口头禅、跳跃想法。
问题2:
  同一句话可能同时包含执行、规则、审计、回滚、触发等多个意图。
问题3:
  系统不能每次都靠人工判断，要自动知道：
    - 该执行
    - 该确认
    - 该熔断
    - 该记录
    - 该学习
问题4:
  UID9622习惯是跳跃式输入，系统必须允许跳跃，但不能乱跑。
问题5:
  Notion、Claude、CNSH、GitHub Action 后续都要看得懂。

⸻

1.2 本页最终输出

输入:
  一句话 / 一个词 / 一段描述 / 一个命令
系统输出:
  Input: 原始输入
  Drawer: 命中的语义抽屉
  Element: 五行映射
  Score: 五行评分
  State: 状态机节点
  Action: 下一步动作
  Audit: 审计策略
  Learn: 是否写入学习

三、核心概念表

3.1 语义抽屉

定义:
  把自然语言中的意图归入稳定分类。
示例:
  “不许动这个” → RULE
  “帮我跑起来” → EXEC
  “自动触发” → HOOK
  “先回滚” → BREAK
  “查一下记录” → AUDIT
  “DNA绑定” → TRACE

⸻

3.2 五行映射

定义:
  把抽屉类型映射为五行属性，用五行关系判断系统动作。
五行定位:
  金: 规则 / 审计 / 熔断
  水: DNA / 追溯 / 记忆
  木: 执行 / 自动 / Hook
  火: 价值 / 表达 / 文明
  土: 系统 / 资源 / 承载

⸻

3.3 状态机

S1: 输入记录 / DNA绑定
S2: 语义解析
S3: 路由分发
S4: 规则确认 / 冲突裁决
S5: 指令确认
S6: 执行
S7: 审计闭环
S8: 熔断 / 回滚

⸻

四、抽屉与五行映射主表

抽屉类型	中文含义	五行	系统含义	默认状态
RULE	规则 / 禁止 / 红线	金	规则优先	S4 / S8
AUDIT	审计 / 校验 / 对账	金	检查与留痕	S7
BREAK	熔断 / 回滚 / 终止	金	停止或恢复	S8
TRACE	DNA / 追溯 / 签名	水	记录来源	S1
EXEC	执行 / 跑 / 搞 / 上线	木	实际动作	S6
HOOK	自动 / 触发 / 一键	木	自动链路	S3
VALUE	价值 / 文明 / 底色	火	价值判断	S4
EXPR	表达 / 对外 / 文案	火	输出表达	S2 / S6
SYS	系统 / 环境 / 资源	土	承载环境	S2 / S3
RESOURCE	资源 / 算力 / 队列	土	调度资源	S3

⸻

五、五行决策规则

5.1 五行相生

金生水:
  规则产生记录。
  例: 红线命中后必须写入DNA日志。
水生木:
  追溯支持执行。
  例: DNA确认后允许进入执行。
木生火:
  执行产生价值表达。
  例: 跑完后生成报告、文案、页面。
火生土:
  价值沉淀为系统结构。
  例: 表达变成模板、数据库、规则页。
土生金:
  系统承载反哺规则。
  例: 执行环境稳定后形成铁律。

⸻

5.2 五行相克

金克木:
  规则限制执行。
  例: 禁止项出现，不允许直接跑。
木克土:
  执行消耗资源。
  例: 大量任务进入资源调度。
土克水:
  环境限制记录。
  例: 本地环境不完整，暂缓追溯写入。
水克火:
  记录压制表达。
  例: 涉及隐私时，不允许直接对外发布。
火克金:
  情绪或价值表达冲击规则。
  例: 激烈表达触发规则确认。

⸻

5.3 默认决策逻辑

规则1:
  只要金强且涉及RULE/BREAK，优先进入S8或S4。
规则2:
  木强且无金克制，进入S6执行。
规则3:
  水强时，先进入S1记录，再进入下一步。
规则4:
  火强时，先判断是否涉及公开表达或价值红线。
规则5:
  土强时，先检查资源、环境、依赖是否齐全。
规则6:
  多五行同时出现时，先查相克，再查相生。
规则7:
  不确定时，不直接执行，进入S4确认。

# 龍魂·语义五行自进化内核.cnsh
# DNA: #CONFIRM🌌9622-ONLY-ONCE🧬LK9X-772Z
词典 = {
  "RULE": ["不许", "禁止", "不可", "绝对不", "不能动", "红线", "铁律", "家法"],
  "BREAK": ["回滚", "熔断", "终止", "停止", "撤回", "复原", "刹车"],
  "HOOK": ["自动", "触发", "一键", "钩子", "Hook", "联动", "链路"],
  "EXEC": ["执行", "搞", "跑", "上线", "落地", "做出来", "推进去"],
  "TRACE": ["DNA", "追溯", "签名", "水印", "确认码", "GPG", "留痕"],
  "AUDIT": ["审计", "校验", "对账", "检查", "复盘", "查一下"],
  "VALUE": ["价值", "人民", "干净", "不跪", "不偷", "底色"],
  "EXPR": ["表达", "翻译", "说人话", "对外解释", "文案"],
  "SYS": ["系统", "资源", "环境", "Notion", "CNSH", "终端"]
}
抽屉五行 = {
  "RULE": "金",
  "AUDIT": "金",
  "BREAK": "金",
  "TRACE": "水",
  "EXEC": "木",
  "HOOK": "木",
  "VALUE": "火",
  "EXPR": "火",
  "SYS": "土",
  "RESOURCE": "土"
}
权重 = {
  "金": 1.0,
  "木": 1.0,
  "水": 1.0,
  "火": 1.0,
  "土": 1.0
}
相生 = {
  "金": "水",
  "水": "木",
  "木": "火",
  "火": "土",
  "土": "金"
}
相克 = {
  "金": "木",
  "木": "土",
  "土": "水",
  "水": "火",
  "火": "金"
}
状态说明 = {
  "S1": "输入记录",
  "S2": "语义解析",
  "S3": "路由分发",
  "S4": "规则确认",
  "S5": "指令确认",
  "S6": "执行",
  "S7": "审计闭环",
  "S8": "熔断回滚"
}
def 抽屉识别(输入):
  命中 = []
  for 抽屉, 词组 in 词典.items():
    for 词 in 词组:
      if 词 in 输入:
        命中.append(抽屉)
        break
  if not 命中:
    命中 = ["SYS"]
  return list(set(命中))
def 五行映射(抽屉):
  return [抽屉五行.get(x, "土") for x in 抽屉]
def 评分(五行):
  分 = {"金": 0, "木": 0, "水": 0, "火": 0, "土": 0}
  for x in 五行:
    分[x] += 权重[x]
  return 分
def 主五行(分):
  return max(分, key=分.get)
def 存在相克(分):
  for a in 分:
    for b in 分:
      if 分[a] > 0 and 分[b] > 0 and 相克.get(a) == b:
        return True, a, b
  return False, None, None
def 存在相生(分):
  for a in 分:
    for b in 分:
      if 分[a] > 0 and 分[b] > 0 and 相生.get(a) == b:
        return True, a, b
  return False, None, None
def 决策(抽屉, 分):
  if "BREAK" in 抽屉:
    return "S8"
  if "RULE" in 抽屉 and "EXEC" in 抽屉:
    return "S4"
  if "RULE" in 抽屉:
    return "S4"
  if "TRACE" in 抽屉 and len(抽屉) == 1:
    return "S1"
  if "HOOK" in 抽屉:
    return "S3"
  克, a, b = 存在相克(分)
  if 克:
    return "S4"
  if "EXEC" in 抽屉:
    return "S6"
  if "AUDIT" in 抽屉:
    return "S7"
  return "S2"
def 审计策略(状态):
  if 状态 == "S8":
    return "FORCE"
  if 状态 == "S4":
    return "STRICT"
  if 状态 == "S7":
    return "LOG"
  return "PASSIVE"
def 风险级别(状态):
  if 状态 == "S8":
    return "极高"
  if 状态 == "S4":
    return "高"
  if 状态 == "S7":
    return "中"
  return "低"
def 学习(输入, 抽屉, 状态):
  for x in 抽屉:
    元 = 抽屉五行.get(x, "土")
    if 状态 == "S6":
      权重[元] += 0.05
    if 状态 == "S4":
      权重[元] -= 0.02
    if 状态 == "S8":
      权重[元] -= 0.05
  词组 = 输入.split()
  for 词 in 词组:
    if len(词) > 1 and 抽屉:
      目标 = 抽屉[0]
      if 目标 not in 词典:
        词典[目标] = []
      if 词 not in 词典[目标]:
        词典[目标].append(词)
def CNSH(输入):
  抽屉 = 抽屉识别(输入)
  五行 = 五行映射(抽屉)
  分 = 评分(五行)
  状态 = 决策(抽屉, 分)
  学习(输入, 抽屉, 状态)
  return {
    "Input": 输入,
    "Drawer": 抽屉,
    "Element": 五行,
    "Score": 分,
    "MainElement": 主五行(分),
    "State": 状态,
    "StateName": 状态说明[状态],
    "Audit": 审计策略(状态),
    "Risk": 风险级别(状态)
  }
