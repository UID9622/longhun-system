# CNSH 本地部署快速指南

## 📋 概述

本指南将帮助您快速部署 CNSH 系统，将 Obsidian 和 Ollama 整合为一个强大的本地知识管理和智能助手系统。

## 🚀 一键安装

### 1. 运行安装脚本

```bash
# 进入项目目录
cd /Users/zuimeidedeyihan/LuckyCommandCenter/cnsh-deployment

# 运行安装脚本
./install.sh
```

安装脚本将自动完成以下步骤：
- 安装系统依赖（Node.js、Python、Git）
- 安装和配置 Ollama
- 下载中文大模型（qwen:7b-chat、chatglm3:6b）
- 安装 CNSH 核心系统
- 准备 Obsidian 插件

### 2. 配置系统

1. **编辑 .env 文件**：
   ```bash
   vim .env
   ```
   配置以下关键项：
   ```env
   OBSIDIAN_VAULT_PATH=/path/to/your/obsidian/vault
   OLLAMA_MODEL=qwen:7b-chat
   ```

2. **启动 CNSH 服务**：
   ```bash
   ./start-cnsh.sh
   ```

3. **验证安装**：
   - 访问 http://localhost:3000
   - 应该看到 CNSH 服务状态页面

## 🔧 手动部署

如果您想了解安装过程的详细步骤，可以参考以下手动部署流程：

### 1. 安装依赖

```bash
# 安装 Node.js
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# 安装 Python 3
sudo apt-get update
sudo apt-get install -y python3 python3-pip

# 安装 Ollama
curl -fsSL https://ollama.ai/install.sh | sh
```

### 2. 下载模型

```bash
# 启动 Ollama 服务
ollama serve &

# 下载中文模型
ollama pull qwen:7b-chat
ollama pull chatglm3:6b
```

### 3. 安装 CNSH 核心

```bash
# 安装 Node.js 依赖
npm install

# 安装 Python 依赖
pip3 install -r requirements.txt

# 创建 .env 配置文件
cp .env.example .env
# 编辑 .env 文件，配置您的设置
```

### 4. 安装 Obsidian 插件

1. 打开 Obsidian
2. 进入设置 → 第三方插件
3. 关闭安全模式
4. 点击 "浏览社区插件"，搜索 "CNSH Connector"
5. 安装并启用插件
6. 在插件设置中配置 CNSH 服务器地址

## 🛠️ 故障排除

### 1. CNSH 服务无法启动

**问题**：`npm start` 失败

**解决方案**：
1. 检查端口 3000 是否被占用
2. 确认所有依赖已正确安装
3. 查看 logs 文件夹中的错误日志

```bash
# 查看错误日志
tail -f logs/cnsh.log

# 尝试重新安装依赖
npm install --force
```

### 2. Ollama 连接失败

**问题**：CNSH 无法连接到 Ollama

**解决方案**：
1. 确认 Ollama 服务正在运行
   ```bash
   pgrep -f "ollama serve"
   ```
2. 检查 Ollama 是否正确安装
   ```bash
   ollama --version
   ```
3. 重新启动 Ollama 服务
   ```bash
   pkill -f "ollama serve"
   nohup ollama serve > /dev/null 2>&1 &
   ```

### 3. 模型未找到

**问题**：提示模型不存在

**解决方案**：
1. 检查已安装的模型
   ```bash
   ollama list
   ```
2. 重新下载模型
   ```bash
   ollama pull qwen:7b-chat
   ```
3. 在 .env 文件中更新模型名称

### 4. Obsidian 插件无法连接

**问题**：Obsidian 插件显示连接错误

**解决方案**：
1. 检查 CNSH 服务是否运行在 http://localhost:3000
2. 确认 API 密钥（如果配置了）
3. 检查防火墙设置
4. 查看浏览器控制台错误信息
