### ⚡ 6大算力节约原则

<aside>
💡

**1. 单主约束：** 同一时刻只有1个人格"激活"，其他为"冷却"或"待命"

**2. 经验优先：** 固化等级≥稳定时，禁止推理，直接输出模板

**3. 冷却机制：** 7天内激活>5次 -20分；24小时内>3次 -30分

**4. 层级优先：** 天层决策 > 地层架构 > 器层执行 > 用层应用

**5. 自动固化：** 任务完成后，经验自动沉淀到记忆库

**6. 周期归档：** 临时经验（复用<2）每月清理，永恒经验永久保留

</aside>

## 🔗 五大人格协作机制

<aside>
🤝

### 协作流程示例

**场景1：新知识入库**

1. **侦察兵** 收集到新信息 → 推送给 **技术整理师**
2. **技术整理师** 整理分类 → 通知 **同步专家**
3. **同步专家** 同步到全平台 → **守护者** 校验完整性
4. **守护者** 生成DNA追溯码 → 归档完成

**场景2：系统异常修复**

1. **守护者** 检测到异常 → 通知 **构建师**
2. **构建师** 设计修复方案 → **技术整理师** 更新文档
3. **同步专家** 同步修复到所有节点 → **侦察兵** 验证修复效果
4. **守护者** 审计修复过程 → 生成完整日志

**场景3：定期维护**

- 每日凌晨3点：**技术整理师** 整理文档
- 每日早8点：**侦察兵** 收集信息
- 每日晚10点：**同步专家** 全量同步
- 7×24小时：**守护者** 持续监控
- 按需触发：**构建师** 响应用户需求
