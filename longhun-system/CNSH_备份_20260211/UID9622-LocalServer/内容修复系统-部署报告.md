## 🎯 核心功能

### 1. 智能平台检测

系统可以自动识别内容来源：

| 平台 | 特征 | 检测结果 |
|------|------|----------|
| Notion | `{{notion-}}` | ✅ 可识别 |
| Claude | `{{URL-}}` | ✅ 可识别 |
| ChatGPT | 不完整代码块 | ✅ 可识别 |
| 通用问题 | 格式错误 | ✅ 可识别 |

### 2. 问题修复能力

| 问题类型 | 示例 | 修复方式 | 状态 |
|---------|------|---------|------|
| 断裂链接 | `[文字](http://system.sh)` | 转换为 `[文字]` | ✅ 已实现 |
| 代码块不完整 | 缺少结束 ``` | 自动补全 | ✅ 已实现 |
| 列表格式错误 | `-item` | 修复为 `- item` | ✅ 已实现 |
| Notion压缩链接 | `{{notion-123}}` | 转换为提示 | ✅ 已实现 |
| 特殊字符 | 零宽字符 | 自动清理 | ✅ 已实现 |

### 3. 多种使用方式

#### 方式1: 快速启动脚本 ⭐ 推荐

```bash
# 查看帮助
bash quick_repair.sh --help

# 修复文件
bash quick_repair.sh test.txt

# 从剪贴板修复
bash quick_repair.sh --clipboard
```

#### 方式2: 直接调用脚本

```bash
# 修复文件
python3 UID9622-Scripts/content_repair.py test.txt

# 从剪贴板（需要pyperclip）
bash UID9622-Scripts/repair_from_clipboard.sh
```

#### 方式3: API调用

```bash
curl -X POST http://127.0.0.1:9622/api/repair_content \
  -H "Content-Type: application/json" \
  -d '{"content":"要修复的内容"}'
```
