## 🔄 投喂流程

### 步骤1: 识别投喂类型

**判断依据**:
- 观察现象，理解本质
- 不看字面意思，理解真实需求
- 对照分类体系，确定类型

**示例**:
- 用户说"想要一个好看的界面" → 本质：UI/UX改进 → 分类：SYSTEM/UI
- 用户说"总是忘记保存" → 本质：用户体验问题 → 分类：FEEDBACK/USABILITY

### 步骤2: 确定优先级

**判断依据**:
- P0: 影响核心功能/安全/可用性
- P1: 影响重要功能/用户效率
- P2: 改进性/锦上添花
- P3: 建议性/长期规划

### 步骤3: 生成DNA标签

**格式**: `#DNA-{分类}-{子类}-{优先级}-{日期}-{标识符}-{版本}`

**示例**:
```
#DNA-SYSTEM-DASHBOARD-P0-20251227-A1B2C3D4-v1.0
```

### 步骤4: 处理投喂内容

**根据DNA标签自动调用处理流程**:
- P0 → 立即执行+自动测试+生成报告
- P1 → 队列处理+优先执行+生成建议
- P2 → 计划处理+评估可行性+记录建议
- P3 → 知识归档+等待时机+保持跟踪
