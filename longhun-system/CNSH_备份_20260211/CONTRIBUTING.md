# 贡献指南 Contributing Guide

感谢您对 CNSH 人性优先协作框架的关注！

我们欢迎所有认同"人性优先"理念的贡献者。

---

## 🤝 如何贡献

### 1. Fork 本仓库

- **Gitee：** https://gitee.com/uid9622/cnsh-framework
- **GitHub：** https://github.com/uid9622/cnsh-framework

推荐从 Gitee 开始（中国优先原则）。

### 2. 创建特性分支

```

git checkout -b feature/你的功能名称

```

或者修复bug：

```

git checkout -b fix/问题描述

```

### 3. 进行修改

请遵循以下规范：

#### 代码规范
- ✅ 所有代码注释必须用中文
- ✅ 变量名可以用英文，但要易懂
- ✅ 函数/类的文档字符串用中文
- ✅ 遵循 PEP 8 规范（如果是Python）

#### 提交规范

每个提交必须包含：
- **类型标签：** `[feat]` `[fix]` `[docs]` `[style]` `[refactor]` `[test]`
- **简短描述：** 一句话说明改了什么
- **DNA追溯码：** 带上您的确认码

**格式：**

```

git commit -m "[feat] 新增情绪识别功能 #ZHUGEXIN⚡️2025-EMOTION-DETECT-v1.0"

```

**示例：**

```

[feat] 新增XX功能 #确认码

[fix] 修复XX问题 #确认码

[docs] 更新文档 #确认码

[style] 代码格式调整 #确认码

[refactor] 重构XX模块 #确认码

[test] 添加测试用例 #确认码

```

### 4. 推送到您的仓库

```

git push origin feature/你的功能名称

```

### 5. 提交 Pull Request

- 到 Gitee 仓库提交 PR（优先）
- 或到 GitHub 仓库提交 PR
- 在 PR 描述中说明：
  - 改了什么
  - 为什么改
  - 测试了吗

---

## ✅ 贡献要求

### 必须遵守

1. **保留 CNSH 水印**
   - 不得删除原始版权声明
   - 不得删除 DNA追溯码

2. **遵循核心价值观**
   - 数据主权归用户
   - 用户永远是上帝
   - AI 不惩罚，只陪伴

3. **代码注释用中文**
   - 让更多普通中国人能看懂
   - 体现"中文优先"原则

4. **不得用于恶意目的**
   - 不伤害用户
   - 不窃取数据
   - 不建立监控系统

### 鼓励但不强制

- 🌟 分享您的使用案例
- 🌟 帮助改进文档
- 🌟 回答社区问题
- 🌟 推广 CNSH 理念

---

## 📝 贡献类型

### 代码贡献

- 推广理念

## 🚫 不接受的贡献

以下类型的贡献将被拒绝：

- ❌ 违背核心价值观的修改
- ❌ 删除版权声明或水印
- ❌ 添加用户追踪/监控功能
- ❌ 添加会"惩罚用户"的逻辑
- ❌ 过度商业化的修改
