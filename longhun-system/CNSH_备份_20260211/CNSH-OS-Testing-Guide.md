# UID9622 · CNSH-OS 测试指南

## 🧪 全链路测试流程

本指南详细说明如何测试CNSH-OS全链路系统的各个组件，确保"人格→任务→决策→审计"流程正常运行。

## 📋 测试准备

### 1. 导入模板数据

1. 在Notion中创建四个数据库：
   - Personas (12核心人格)
   - Tasks (任务流水线)
   - Decision Records (决策记录)
   - Audit Loop (审计闭环)

2. 导入CSV模板：
   - CNSH-OS-Personas-Database-Template.csv
   - CNSH-OS-Tasks-Database-Template.csv
   - CNSH-OS-Decision-Database-Template.csv
   - CNSH-OS-Audit-Database-Template.csv

3. 设置Relation字段：
   - 按照CNSH-OS-Notion-Relations-Guide.md中的说明建立所有关联

### 2. 添加公式字段

根据CNSH-OS-Formula-Templates.md中的公式，为各数据库添加对应的公式字段。

## 🔍 测试场景

### 场景1：基础数据流测试

**目标**：验证基础数据在各数据库间的流通

**测试步骤**：
1. 在Tasks数据库创建新任务"测试-001"
2. 设置识别人格为"文心"，执行人格为"执行官"
3. 在Decision Records数据库创建对应决策记录"Decision-测试-001"
4. 关联Tasks记录"测试-001"和参与人格"文心、执行官"
5. 在Audit Loop数据库创建审计记录"Audit-测试-001"
6. 关联Tasks记录"测试-001"、Decision记录"Decision-测试-001"和审计人格"上帝之眼"

**预期结果**：
- 所有关联关系正确建立
- 在任一数据库都能查看相关联的记录
- 公式字段正确计算

### 场景2：任务自动化测试

**目标**：验证任务自动分配和状态更新

**测试步骤**：
1. 创建任务类型为"建设"的新任务
2. 验证系统推荐的人选是否为"执行官, 工具师"
3. 更新任务状态为"进行中"
4. 验证完成度公式是否正确更新为0.5
5. 更新任务状态为"已完成"
6. 验证完成度公式是否正确更新为1

**预期结果**：
- 人格推荐公式工作正常
- 状态变化触发公式正确计算
- 任务完成度与状态保持一致

### 场景3：决策流程测试

**目标**：验证决策流程和风险评估

**测试步骤**：
1. 创建高风险决策记录
2. 设置风险标签为"高"
3. 验证风险评分公式是否返回90
4. 关联多个任务（3个以上）
5. 验证影响范围评估是否返回"系统级"

**预期结果**：
- 风险评分根据风险标签正确计算
- 影响范围根据关联任务数量正确评估

### 场景4：审计闭环测试

**目标**：验证审计流程和回滚机制

**测试步骤**：
1. 创建审计记录，风险标签设为"高"
2. 添加冲突发现内容
3. 验证审计紧急度公式是否返回95
4. 更新审计结论为"需要重新设计"
5. 验证审计状态是否更新为"需重做"

**预期结果**：
- 审计紧急度根据风险标签和冲突情况正确计算
- 审计状态与审计结论保持一致

### 场景5：CNSH伦理约束测试

**目标**：验证CNSH伦理框架和UID9622数据主权协议

**测试步骤**：
1. 创建包含"外部传输"的任务
2. 验证UID9622数据主权检查公式是否返回"⚠️ 可能违反数据主权"
3. 创建包含"评判"、"说教"的内容
4. 验证CNSH人性优先验证公式是否返回"❌ 违反人性优先"
5. 创建纯英文内容
6. 验证中文语义优先检查公式是否返回"❌ 建议增加中文内容"

**预期结果**：
- 所有伦理约束公式按预期工作
- 违反CNSH原则和UID9622协议的内容被正确标记

**目标**：验证基于输入的人格自动激活机制

## 🔧 故障排除

### 常见问题1：Relation字段无法建立

**症状**：无法建立两个数据库之间的关联

**解决方案**：
1. 确认目标数据库已存在
2. 检查数据库名称拼写
3. 确保有足够的权限

### 常见问题2：公式字段计算错误

**症状**：公式字段显示错误或不计算

**解决方案**：
1. 检查公式语法
2. 确认引用的字段名正确
3. 验证字段类型匹配

### 常见问题3：数据流中断

**症状**：数据无法在数据库间正确流通

**解决方案**：
1. 检查Relation字段设置
2. 验证关联字段正确填充
3. 确认数据库过滤条件
