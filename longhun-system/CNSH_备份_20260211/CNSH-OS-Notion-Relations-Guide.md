- 记忆与记录层：档案官、叙事者、历史守望者

### 2. Tasks 数据库 (任务流水线)

### 3. Decision Records 数据库 (决策记录)

### 4. Audit Loop 数据库 (审计闭环)

## 🔗 Relation映射详表

### Tasks 数据库关联

| 字段名 | 类型 | 关联对象 | 说明 |
|--------|------|----------|------|
| 识别人格 | Relation | Personas | 选择1-3个人格用于语义识别、分析输入 |
| 执行人格 | Relation | Personas | 分配任务执行角色 |
| 关联证据/决策 | Relation | Decision Records | 每个任务对应决策记录 |

### Decision Records 数据库关联

| 字段名 | 类型 | 关联对象 | 说明 |
|--------|------|----------|------|

| 关联任务 | Relation | Tasks | 本决策对应的任务 |
| 审计记录 | Relation | Audit Loop | 决策通过后触发审计闭环记录 |

### Audit Loop 数据库关联

| 字段名 | 类型 | 关联对象 | 说明 |
|--------|------|----------|------|
| 审计人格 | Relation | Personas | 谁来执行审计 |
| 关联任务 | Relation | Tasks | 审计对应的任务 |
| 关联决策 | Relation | Decision Records | 审计对应的决策记录 |

### Personas 数据库关联

| 字段名 | 类型 | 关联对象 | 说明 |
|--------|------|----------|------|
| UID9622 DNA | Rich Text | - | 该人格继承信息与核心向量 |
| 触发器 | Relation | Audit Loop/Tasks | 可主动触发事件 |
| 职责 | Relation | Tasks | 该人格负责执行的任务集合 |

## 🛠 Notion操作步骤

### 步骤1：创建Relation字段

1. **Tasks数据库设置**：
   - 打开Tasks数据库 → "+ Add property" → 选择Relation → 关联到Personas
   - 创建名为"识别人格"的Relation字段
   - 重复操作创建"执行人格"Relation字段
   - 创建"关联证据/决策"Relation字段，关联到Decision Records

2. **Decision Records数据库设置**：
   - 创建"参与人格"Relation字段，关联到Personas
   - 创建"关联任务"Relation字段，关联到Tasks
   - 创建"审计记录"Relation字段，关联到Audit Loop

3. **Audit Loop数据库设置**：
   - 创建"审计人格"Relation字段，关联到Personas
   - 创建"关联任务"Relation字段，关联到Tasks
   - 创建"关联决策"Relation字段，关联到Decision Records

4. **Personas数据库设置**：
   - 创建"触发器"Relation字段，关联到Audit Loop/Tasks
   - 创建"职责"Relation字段，关联到Tasks

### 步骤2：添加公式字段（可选）

1. **Tasks数据库公式**：
   ```
   // 自动更新时间
   now()
   
   // 回写人格（如果想记录上一次完成的人格）
   last(执行人格)
   ```

2. **Decision Records数据库公式**：
   ```
   // 风险评分自动填充
   if(Risk_Level=="高", 90, if(Risk_Level=="中", 50, 10))
   ```

### 步骤3：验证关联

1. **测试数据创建**：
   - 在Tasks数据库创建测试任务"T-TEST"
   - 选择识别人格和执行人格
   - 创建对应的Decision Records记录
   - 创建对应的Audit Loop记录

2. **数据流验证**：
   - 检查Tasks能否正确关联到Personas
   - 检查Decision Records能否正确关联到Tasks和Personas
   - 检查Audit Loop能否正确关联到所有相关表
   - 验证全链路数据是否正确显示：Tasks → Decision → Audit → Personas

## 🔧 自动化工作流

### 任务触发流程

1. **任务创建**：
   - Lucky创建任务
   - 系统自动识别任务类型和触发人格
   - 分配执行人格

2. **决策过程**：
   - 关键任务自动创建Decision Records
   - 召集相关人格进行决策
   - 记录决策结果

3. **审计闭环**：
   - 决策通过后自动触发审计
   - 审计人格执行审计流程
   - 记录审计结果和改进建议

4. **系统回写**：
   - 执行结果回写到Tasks
   - 更新Personas的职责和触发器
   - 记录到历史库
