## 📖 系统简介

### 什么是龍魂自动路由系统？

```
路由系统 = 文档自动导航 + 模板智能匹配 + DNA格式自动选择

就像：
  GPS导航 → 自动规划路线
  高速路标 → 告诉你去哪里走哪条路
  智能助手 → 3秒决定用什么模板

龍魂自动路由系统让你：
  ✅ 不用思考"用什么格式"
  ✅ 不用记忆"DNA怎么写"
  ✅ 不用担心"格式不统一"
  ✅ 3秒开始，专注内容创作
```

### 核心价值

```yaml
价值1：标准化
  - 所有文档格式统一
  - 所有DNA追溯码规范
  - 就像公司logo一样专业

价值2：自动化
  - 自动选择模板
  - 自动生成DNA格式
  - 自动记录时间

价值3：可追溯
  - 每个文档有DNA身份证
  - 可以追溯来源和历史
  - 防篡改、防盗用

价值4：效率提升
  - 3秒开始创作
  - 不用重复劳动
  - 专注内容，不纠结格式
```

## 🚦 DNA格式自动选择规则

### 规则1：按受众分类

```yaml
受众是自己/团队 → #ZHUGEXIN⚡️
  特征：
    - 内部使用
    - 不对外公开
    - 团队协作
  
  示例：
    #ZHUGEXIN⚡️-2026-01-26-设计方案-v1.0
    #ZHUGEXIN⚡️-2026-01-26-会议记录-001

受众是公众 → #龍芯⚡️
  特征：
    - 对外发布
    - 公开传播
    - 品牌展示
  
  示例：
    #龍芯⚡️-2026-01-26-用户手册-v1.0
    #龍芯⚡️-2026-01-26-技术文章-v1.0
```

---

### 规则2：按阶段调整

```yaml
草稿阶段 → #ZHUGEXIN⚡️
  - 还在写，不确定要不要公开
  - 先用内部格式
  
发布阶段 → 评估是否改为 #龍芯⚡️
  - 如果要对外发布 → 改为 #龍芯⚡️
  - 如果只是内部正式版 → 保持 #ZHUGEXIN⚡️
```

---

### 规则3：命名规范

```yaml
格式：
  #标识符⚡️-YYYY-MM-DD-主题-版本/序号

标识符：
  ZHUGEXIN = 内部文档
  龍芯 = 公开文档

日期：
  YYYY-MM-DD（年-月-日）
  例：2026-01-26

主题：
  - 用中文或拼音
  - 简短明确
  - 不超过20字符
  
版本/序号：
  v1.0, v1.1, v2.0（正式文档）
  001, 002, 003（临时文档）
```

## 🎬 标准操作流程（SOP）

### SOP1：创建新文档

```yaml
步骤1：打开龍魂文档库
  位置：Notion → 龍魂文档库
  快捷键：Cmd+P → 输入"龍魂文档库"

步骤2：点击New（新建）
  位置：数据库右上角或底部
  快捷键：无（需点击）

步骤3：选择模板
  决策：
    - 内部文档 → "内部模板"
    - 公开文档 → "公开模板"
    - 快速笔记 → "简化模板"

步骤4：填写必填字段
  必填：
    ✅ 标题
    ✅ DNA追溯码（更新日期和主题）
    ✅ 文档状态（默认：草稿）
    ✅ 机密级别
  
  选填：
    主题标签
    版本号
    协作者

步骤5：编写内容
  工具：
    /heading 1-3（标题）
    /bullet（列表）
    /toggle（折叠块）
    /code（代码块）
    /divider（分割线）

步骤6：审核发布
  草稿 → 审核中：
    1. 内容写完
    2. 状态改为"审核中"
    3. @协作者 或 审核人
  
  审核中 → 已发布：
    1. 审核通过
    2. 状态改为"已发布"
    3. 如需对外，复制到发布平台

步骤7：归档（可选）
  已发布 → 已归档：
    1. 文档过时或不再使用
    2. 状态改为"已归档"
    3. 不删除，保留历史
```

---

### SOP2：文档转换（内部→公开）

```yaml
步骤1：评估是否适合公开
  检查清单：
    ☐ 内容无敏感信息
    ☐ 技术方案已成熟
    ☐ 可以对外展示
    ☐ 有宣传价值

步骤2：复制文档
  操作：
    右上角"..." → Duplicate
  
  重命名：
    原：XXX设计方案（内部）
    新：XXX设计方案（公开版）

步骤3：修改DNA追溯码
  原：#ZHUGEXIN⚡️-2026-01-26-设计方案-v1.0
  新：#龍芯⚡️-2026-01-26-设计方案-v1.0

步骤4：调整内容
  删除：
    - 内部讨论
    - 敏感数据
    - 团队成员名字（如不方便公开）
  
  添加：
    - 对外说明
    - 使用示例
    - 联系方式

步骤5：更新属性
  机密级别：机密 → 公开
  文档状态：已发布
  发布平台：添加发布渠道

步骤6：发布
  导出：
    右上角"..." → Export → Markdown
  
  发布到：
    GitHub / CSDN / 知乎等
  
  填写：
    文档链接字段
```

### SOP3：定期审计

```yaml
频率：每月1次

步骤1：筛选过期文档
  筛选条件：
    - 最后更新 > 90天
    - 状态 = 已发布
```

## ⚠️ 常见错误与解决

### 错误1：忘记添加DNA追溯码

```yaml
症状：
  文档创建了，但没有DNA追溯码

解决方法：
  1. 打开文档
  2. 在第一行添加：
     **DNA追溯码：** #ZHUGEXIN⚡️-YYYY-MM-DD-主题-版本
  3. 保存

预防：
  使用模板创建文档（自动包含）
```

---

### 错误2：DNA格式错误

```yaml
常见错误：
  ❌ #ZHUGEXIN-2026-01-26-设计-v1.0（少了⚡️）
  ❌ #龍芯 2026-01-26-设计-v1.0（少了⚡️和连字符）
  ❌ #ZHUGEXIN⚡️20260126设计v1.0（格式混乱）

正确格式：
  ✅ #ZHUGEXIN⚡️-2026-01-26-设计-v1.0
  ✅ #龍芯⚡️-2026-01-26-设计-v1.0

解决方法：
  复制粘贴标准格式，只改日期和主题
```

---

### 错误3：内部文档意外公开

```yaml
症状：
  内部文档被分享到外部

解决方法：
  1. 立即撤回分享链接
  2. 检查文档内容是否有敏感信息
  3. 如需对外，转换为公开版本
  4. 修改DNA追溯码

预防：
  1. 设置机密级别
  2. 限制访问权限
  3. 分享前二次确认
```

---

### 错误4：版本号混乱

```yaml
症状：
  同一文档有多个版本，不知道哪个是最新

解决方法：
  1. 确定最新版本
  2. 在标题添加"（最新版）"
  3. 旧版本改为"（归档）"
  4. 在最新版添加链接到旧版本

预防：
  1. 使用版本号：v1.0, v1.1, v2.0
  2. 不要复制粘贴，用"Duplicate"
  3. 定期清理旧版本
```

---

### 错误5：协作冲突

```yaml
症状：
  两个人同时编辑，内容冲突

解决方法：
  1. Notion会自动合并
  2. 如有冲突，查看历史记录
  3. 手动合并冲突内容

预防：
  1. 编辑前@通知协作者
  2. 使用评论功能讨论
  3. 避免同时编辑同一段
```
