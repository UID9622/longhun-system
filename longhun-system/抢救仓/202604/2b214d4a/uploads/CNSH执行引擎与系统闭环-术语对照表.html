🔐 第四层 · 确认机制（Confirm Model）· 关键路径只有你能按
核心原则：关键决策必须确认，流程执行可以自动。不是每一步都停——而是关键节点才停。你做的不是"确认机制"，是唯一主权确认锚点。

决策确认
最核心		 是否执行、是否发布、是否升级层级 不是每一步都打CONFIRM！

如果全流程强制确认：
• 没有流动
• 没有自动化
• 每一步都卡你
• 系统无法成长

正确结构：
低风险 → 自动流动
中风险 → 路由确认

🔢 第六层 · 状态机（State Machine）· 系统现在是哪一步
一句话：状态机决定一条输入在系统里当前处于哪一步，以及下一步只能去哪里。没有状态机，系统会：重复执行、乱跳、卡死、死循环。
状态码你的命名行业术语大白话谁决定进这个状态
S0 INPUT_RECEIVED
已接收		 Initial State 用户一句话刚进入系统。接住输入+生成时间+分配上下文。不能直接从S0去执行。 Input Gateway
S1 RAW_RECORDED
已原始记录		 Logged State 原句已被保留。DNA占位。没有S1，后面一切视为不成立。铁律。 Raw Memory
S2 ROUTED
已路由		 Dispatched State 类型、意图、层级、可见性、去向判定全部完成。输出RouteResult。 Router Core
S3 PLANNED
已拆解计划		 Planned State 子任务、顺序、依赖、执行建议全部拆好。注意：只是计划完成，不是执行。 Task Planner
S4 WAIT_CONFIRM
等待确认		 Approval Pending 进入确认等待区。只有R0（你）能解锁。Tier≥L2、RULE/EXEC/PUBLIC类型、高风险都会到这里。 ACL + Confirm 判定
S5 EXECUTING
执行中		 Running State 执行器已经开始干活。最容易乱跑的状态——必须有超时、错误、回滚保护。 Executor Pool
S6 DONE
已完成		 Completed State 执行完成+回单已生成。结果摘要、执行路径、状态、下一步建议全部输出。 Engine + Audit
S7 FROZEN
已冻结		 Suspended State 不是失败，是挂起。未成熟概念、高敏内容、暂不适合公开/执行的东西先放这里。内容不删，但先不继续。 Router / Guard
S8 FUSED
