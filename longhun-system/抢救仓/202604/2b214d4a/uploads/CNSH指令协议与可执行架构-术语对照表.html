📡 第一层 · 指令协议（Command Protocol）· 系统的「通用语言」
一句话：所有模块之间说话都用同一种格式——结构化JSON，不允许纯文本乱传。机器看得懂、追溯查得到、谁说的一清二楚。
你的命名行业标准术语大白话 + 你可能说的词在系统里的位置/字段
协议①
外层公共头
Header		 Protocol Header
Message Envelope
Metadata Layer		 一句话：信封——每条指令的"身份证"。

你可能说的词：DNA、时间戳、谁发的、版本号

大白话：就像寄快递，箱子外面贴的那张面单：寄件人是谁（role）、单号多少（dna）、什么时候寄的（timestamp）、这条快递属于哪个对话链（session_id）。所有指令必须带，缺一不可。		 dna 全局唯一ID
格式：CNSH-{时间戳}-{随机8位}
timestamp Unix毫秒时间戳
session_id 会话ID
role 触发角色 R0/R1/R2/R3
version 协议版本
铁律
无Header = 直接S8熔断
协议②
内层业务体
Body		 Message Payload
Request Body
Business Layer		 一句话：包裹里的东西——真正要传的内容。

你可能说的词：输入内容、路由结果、执行参数、回单

大白话：信封（Header）说的是"谁寄的"，Body才是箱子里面装的东西。按指令类型不同，Body装不同的字段。输入指令装原文，路由指令装分流结果，执行指令装执行器参数。		 INPUT text + attachments
ROUTE type/intent/tier/vis/line
PLAN steps/deps/need_confirm
EXEC executor/input_params/timeout
RESULT status/outputs/errors/next
CONFIRM confirm_type/uid/remark
协议③
状态尾
Tail		 Status Footer
State Trailer
FSM Checkpoint		 一句话：快递签收单——现在到哪步了、下一步去哪。

你可能说的词：当前状态、下一步、要不要确认

大白话：每条返回指令末尾都带这个尾巴：我现在是S几、我从S几来的、我下一步只能去S几。状态机的"活记录"——不带Tail的返回=无效。		 current_state S0~S8
previous_state 上一个状态
next_allowed_states 允许跳转列表
need_confirm 是否需R0确认
铁律
无Tail = 返回无效，不入审计
编号协议铁律大白话违反后果
铁①所有跨模块调用必须符合协议格式模块之间说话只能用标准JSON格式，不符合格式的=废话。直接返回 S8 熔断
铁②DNA字段全程链路不变一条指令从进系统到出回单，DNA编号不许改。改了=篡改。直接 S8 熔断
铁③回单必须从协议字段自动提取给用户看的回单从JSON字段自动生成，禁止手动改写核心信息。审计不通过，标记异常

代码①
核心枚举
Core Enums		 Enum Types
Constant Definitions		 把所有"选项"变成代码里的常量。

大白话：Role有4种、State有9种、TaskType有6种、Visibility有4种——全部写死在代码里，系统只认这些值，其他值=无效=熔断。
enum Role { R0, R1, R2, R3 } enum State { S0..S8 } enum TaskType { IDEA, RULE, METHOD, TASK, MEMORY, EXEC } enum Visibility { OPEN, MASK, PRIVATE, FREEZE }
代码②
权限矩阵
Permission Matrix		 RBAC Config
Access Control Map		 4个角色×8种动作，谁能干什么一目了然。

大白话：R0什么都能干，R1能干带星号的要过安全检查，R2只能读写路由执行，R3只能读和路由。直接写死在代码里——不能改、不能绕。
const PERM = { R0: [A1..A8], // 全部 R1: [A1,A2,A3,A4*,A6*,A7*], R2: [A1,A2,A3,A4*], R3: [A1,A2] // 只读+路由 }
代码③
确认触发条件
Confirm Conditions		 Policy Rules
Trigger Predicates		 什么时候需要你按确认键——写成函数。

大白话：三个条件满足任意一个就触发CONFIRM：层级≥L2、类型是规则/执行/公开、风险等级高。代码里用函数数组表示，any一个true就停下来等你。
const CONFIRM_COND = [ (r) => r.tier >= 2, (r) => ['RULE','EXEC','PUBLIC'] .includes(r.type), (r) => r.risk === 'HIGH' ]

🔄 第四层 · 异常重试与回滚（Retry & Rollback）· 出错了怎么办
机制行业术语大白话具体规则
重试
Auto Retry		 Retry Policy
Exponential Backoff		 网络断了、文件读写失败、工具调用异常——自动再试。

大白话：就像手机信号不好发不出消息，自动帮你重新发。最多试2次，每次间隔30秒。2次还不行→升级为🟡二级熔断。		 最多重试 2次
间隔 30秒
重试失败 → 🟡二级挂起

适用：网络超时、文件读写、外部工具异常
回滚
Auto Rollback		 State Rollback
Snapshot Restore		 严重出错——恢复到出错之前的样子。

大白话：就像游戏里"读取存档"——回到没出错之前的状态。所有文件、配置全部恢复。恢复后验证哈希确认"真的回去了"。		 触发条件：
• 🔴一级熔断
• 哈希不匹配/DNA篡改
• UID9622手动触发

步骤：锁定 → 提取快照 → 恢复 → 验证哈希 → 写审计 → 通知UID9622
版本回滚
Version Rollback		 Version Control
Release Rollback		 发现新版有问题——退回到指定旧版本。

大白话：新版本翻车了，不怕——提供旧版的DNA追溯码，自动帮你退回去。退回前先备份当前版本（万一还要回来）。		 前置条件：提供目标版本DNA + 哈希校验一致

步骤：① 备份当前版本（新DNA）→ ② 提取目标版本 → ③ 覆盖 → ④ 校验哈希/GPG/确认码 → ⑤ 写审计 → ⑥ 回滚回单

优先级检索方式行业术语大白话返回规则
P1 最高 精确匹配DNA追溯码 Exact Key Lookup 你知道编号——直接调出来。像图书馆用书号直接找书。 直接返回对应内容
P2 精确匹配关键词标签 Tag-based Search 你记得关键词——按标签搜。像在Notion里按标签筛选。 按时间倒序返回Top5
P3 语义匹配 Semantic Search
Vector Similarity		 你大概记得说了什么——用AI理解意思去找。说"那个关于字体的想法"也能找到。 按相似度排序返回Top3
P4 最低 模糊匹配 Fuzzy Search 实在想不起来——模糊搜。只返回标题和摘要，不给详情。你再挑。 仅返回标题+摘要，不返回详情

• Layer A 私有层内容自动过滤——只返回 Layer B 可检索索引

• 熔断记录、审计日志——仅UID9622可检索
