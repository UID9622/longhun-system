🧬 第一章 · DNA三层身份体系

DNA不是一个码，是三层码。每一层解决不同的问题，三层叠加才是完整身份。

层级 名称 格式 生成时机 用途
L1 路由编号
Routing ID		 UID{4位数字}-{神兽代号}{2位序号}
例: UID9622-LH01		 用户注册/代理创建时
一次生成，永不改变		 永久身份标识。等同身份证号。花名册主键。所有后续DNA的根锚点。
L2 DNA追溯码
DNA Trace Code		 DNA::{类型}-{UID}-{时间戳8位}-{功能标签}-{版本}
例: DNA::SPEC-9622-20260416-FUNC-V1		 每次创作/操作/输出时
事件级，一事一码		 事件印记。谁、什么时候、做了什么、输出了什么。追溯链的核心节点。
L3 确认码
Confirmation Code		 {DNA追溯码}::{HMAC-SHA256前8位}
例: DNA::SPEC-9622-20260416-FUNC-V1::a3f7c2e1		 需要权限验证时
按需生成，可过期		 权限验证。证明"这个操作确实是UID9622授权的"。防伪造、防篡改。
三层关系： L1 路由编号（永久） ─── 人的身份证 │ ├── L2 DNA追溯码（事件级） ─── 每件事的指纹 │ │ │ └── L3 确认码（按需） ─── 授权戳 │ ├── L2 另一个追溯码... │ └── L3 ... └── ... 规则： · L1 全局唯一，创建后不可修改不可回收 · L2 由 L1 派生，包含 L1 的 UID 字段 · L3 由 L2 派生，HMAC密钥仅存于 :9622 服务端 · 任何外部系统只能验证 L3，不能生成 L3

☀️ 第三章 · 阳链功能 — 创建与嵌入

阳链 = 明面上的、可见的、主动的DNA操作。创作者知道自己在做什么。

功能 端点 方法 规格
创建DNA记录
Yang Create		 /yang/create POST 输入参数：
· uid *必填 — 创作者UID（4位数字）
· type *必填 — DNA类型（ART/SPEC/AGENT/AUDIT/CONF/SYS/KNOW/FLOW）
· tag *必填 — 功能标签（大写英文+数字，≤12字符）
· version 选填 — 版本号，默认 V1
· meta 选填 — JSON对象，附加元数据（标题、摘要、关联DNA等）
输出：
· dna_code — 生成的完整DNA追溯码
· confirm_code — L3确认码（含HMAC签名）
· created_at — ISO 8601时间戳
· chain_id — 阳链存储序号
规则：
· 同一UID+同一TAG+同一日期：自动追加版本号（V1→V2→V3...）
· DNA码一旦创建不可删除、不可修改（只能追加新版本）
· 每次创建自动写入审计日志
嵌入DNA水印
Yang Embed		 /yang/embed POST 输入参数：
· dna_code *必填 — 要嵌入的DNA追溯码
· content *必填 — 原始内容（文本/Base64图片/HTML）
· mode 选填 — 嵌入模式：
  steganography 隐写术（图片，像素级不可见）
  watermark 文字水印（文本，零宽字符嵌入）
  metadata 元数据注入（HTML/文档头部）
  默认：根据content类型自动选择
输出：
· embedded_content — 嵌入DNA后的内容
· embed_method — 实际使用的嵌入方式
· verify_hash — 校验哈希（用于后续提取验证）
嵌入技术规格：
· 隐写术：LSB（最低有效位）嵌入，每像素修改≤1bit，肉眼不可见
· 零宽字符：使用U+200B/U+200C/U+200D/U+FEFF编码DNA到文本间隙
· 元数据：HTML注释 + meta标签 + 自定义data属性三重嵌入
· 所有模式都不影响原始内容的显示效果和使用体验
⚠️ 铁律：嵌入操作不可逆。一旦DNA嵌入内容，该DNA与该内容永久绑定。如需更新DNA，只能创建新版本重新嵌入。
🌙 第四章 · 阴链功能 — 追溯与提取

阴链 = 暗面的、被动的、追查的DNA操作。当内容在外部流转时，阴链追踪它的足迹。

功能 端点 方法 规格
DNA追溯查询
Yin Trace		 /yin/trace/{dna} GET 输入：
· {dna} 路径参数 — 完整DNA追溯码或L1路由编号
· ?depth=full|summary 查询参数 — 追溯深度
· ?from=YYYYMMDD&to=YYYYMMDD 时间范围
输出（depth=full）：
· origin — 创建者信息（UID、神兽归属、信任等级）
· timeline — 完整时间线（创建→嵌入→传播→检测记录）
· derivatives — 衍生品列表（谁引用/转载/修改了这个DNA）
· audit_history — 三色审计历史
· credit_impact — 该DNA对知行积分的影响
输出（depth=summary）：
· owner / created / status / spread_count
规则：
· 用L1查询：返回该用户所有DNA记录
· 用L2查询：返回单个事件的完整追溯链
· 查询本身也写入审计日志（谁在查谁）
· 公开层DNA任何人可查；代理层需代理权限；私密层只有本人
DNA提取
Yin Extract		 /yin/extract POST 输入：
· content *必填 — 待检测内容（文本/Base64图片/HTML/URL）
· mode 选填 — 检测模式：all（默认）/ steganography / watermark / metadata
输出：
· found — boolean，是否找到DNA
· dna_codes[] — 提取到的所有DNA码列表
· embed_methods[] — 每个码的嵌入方式
· integrity — 完整性校验结果（match/tampered/partial）
· confidence — 置信度（0-100%）
规则：
· 三种嵌入方式全部扫描，交叉验证
· integrity=tampered 时触发🟡三色预警
· 置信度<50% 记录但不作为追溯依据

🔥 第十章 · 数字根×五行×熔断对照

每个DNA追溯码都有数字根。数字根决定五行归属，五行归属决定龍魂层级，部分数字根触发自动熔断。

数字根 五行 龍魂层 熔断 规则说明
1, 6 水 💧 L1 记忆 无 存东西的层。DNA存储、记忆压缩、知识库写入。水主收纳，不熔断。
2, 7 火 🔥 L2 文明 无 照亮的层。审计、追溯、证据链、维权。火主照见，不熔断。
3, 8 木 🌿 L4 创新 dr=3 熔断 长的快的层。创作、实验、新功能。木主生长，但 dr=3 触发熔断——长太快会疯，需要审核拦一下。dr=8通行。
4, 9 金 ⚔️ L0 规则 dr=9 熔断 硬的层。铁律、系统边界、权限控制。金主刚硬，但 dr=9 触发熔断——规则层的变动必须人工确认，自动操作全拦。
5, 0 土 🏔️ L3 普惠 无 装所有的层。公开知识库、社区内容、代理模板。土主承载，不熔断。
⚠️ 熔断铁律：dr=3（木·创新失控）和 dr=9（金·规则篡改）自动触发熔断。其他数字根全部通行。
数字根计算方式：取DNA追溯码中UID+日期的所有数字，逐位相加，循环直到单位数。
例：UID9622 + 20260416 → 9+6+2+2+2+0+2+6+0+4+1+6 = 40 → 4+0 = 4（金·L0·dr≠9 → 通行）
数字根计算流程： DNA::SPEC-9622-20260416-FUNC-V1 │ │ ▼ ▼ UID=9622 DATE=20260416 │ │ └──合并───┘ │ 9+6+2+2+2+0+2+6+0+4+1+6 = 40 │ 4+0 = 4 │ dr=4 → 金 → L0规则 → 非9 → 通行 ✅ 如果 dr=3 → 木·创新层 → 🔴自动熔断 → 需人工审核 如果 dr=9 → 金·规则层 → 🔴自动熔断 → 需人工确认
