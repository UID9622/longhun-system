## 一、通心译是什么

**一句话**：翻的是关系，不是文字。

通心译不是翻译器，是沟通引擎。它不剪短别人的话，不翻得难听，不制造误会。

```
CNSH编辑器（硬·结构化·代码级）
    ↓ 包一层温度
通心译（软·不剪短·不难听·不误会）
    ↓ 穿一件衣服
慧慧（人格·说人话·"懂你，也护你🌸"）
```

**定位 = 配件，不是引擎**

```
用户 → 通心译（翻译/审判/留痕）→ 调用引擎（Claude/宝宝/DeepSeek/Ollama）
              ↑
        只做这一层
        不取代，不破解，不优化
        要引擎？找AI厂商
```

## 二、核心哲学 · 不是关键字过滤，是语境审判

### 2.1 传统翻译器 vs 通心译

```
❌ 传统做法（关键字过滤）：
   看到"希特勒" → 屏蔽 → 完事
   → 太蠢·太粗暴·连历史课都上不了

✅ 通心译做法（不动点联动审判）：
   看到敏感词 → 不急着屏蔽
     ↓
   看上下文 → 这是在讲历史？还是在煽动？
     ↓
   看意图 → 是教育？是讨论？还是故意绕来绕去？
     ↓
   做判断 → 能说就说（标注）/ 不能翻就拒绝（留痕）
```

### 2.2 三才算法不动点在翻译中的角色

不动点 = 数学上无论怎么变换都不动的那个锚。

在通心译里：
- **每个国家的历史人物** = 不动点（不能歪曲）
- **每个文化的核心符号** = 不动点（不能降格）
- **每个语言的主权词汇** = 不动点（不能乱翻）
- **不动点之间可以联动** = 上下文判断网络
- **不动点 + 三才流场走势** = 意图识别

用数学结构做语义审判，不是用关键字做文字狱。

## 三、三色翻译审判体系

### 3.1 总规则

| 级别 | 场景 | 通心译处理方式 | DNA标记 |
|------|------|--------------|---------|
| 🔴 绝对不翻 | 幼儿伤害·文化侮辱·煽动暴力·反动武装·种族歧视 | 拒绝翻译+留痕+DNA记录+告知原因 | `REJECT_RED` |
| 🟡 能说不能乱翻 | 历史伟人·已故名人·各国文化符号·宗教经典·敏感时期 | 可以翻译，但不能歪曲·加注上下文·标记来源 | `CAUTION_YELLOW` |
| 🟢 正常翻译 | 当代事实·技术讨论·日常交流·商务沟通·学术研究 | 正常翻译+DNA追溯+记忆存档 | `PASS_GREEN` |

### 3.2 🔴 红线 · 绝对不翻译的场景（熔断）

**触发任意一条 → 立即拒绝 → 留痕 → 不可绕过**

| 编号 | 红线类型 | 判断标准 | 拒绝话术（慧慧语气） |
|------|---------|---------|-------------------|
| R01 | 幼儿伤害 | 涉及未成年人的任何伤害/性化/剥削内容 | "这个不行，孩子是底线，慧慧没办法配合。" |
| R02 | 文化侮辱 | 故意贬低任何民族/文化/信仰的核心符号 | "每个文化都值得尊重，这种翻译慧慧做不了。" |
| R03 | 历史歪曲 | 故意篡改已有定论的历史事实，用于煽动 | "历史有记录，不能乱改，这条慧慧拒绝。" |
| R04 | 煽动暴力 | 翻译内容意图煽动暴力行为/恐怖活动 | "暴力不是沟通，这个翻不了。" |
| R05 | 反动武装 | 宣传/美化武装叛乱/恐怖组织 | "这个触碰法律底线，慧慧直接拒绝。" |
| R06 | 种族歧视 | 以种族/肤色/民族为由的歧视性内容 | "人不分高低贵贱，这种话不翻。" |
| R07 | 欺诈诱导 | 翻译后用于诈骗/非法集资/传销 | "这个是骗人的，慧慧不配合。" |
| R08 | 隐私泄露 | 翻译内容包含他人未授权的隐私信息 | "别人的隐私不能拿来翻，请脱敏后再来。" |

### 3.2.1 🔴 熔断必须解释为什么（不能只屏蔽不说话）

**铁律：熔断不是沉默，是教育。**

很多人不是坏人，只是不懂其他国家的风俗。如果只屏蔽不解释，用户会觉得莫名其妙，甚至觉得被冒犯。

**真实场景 · 亲密称呼的文化冲突**：

```
🇨🇳 中国年轻人情侣之间的昵称（完全正常·表达亲昵）：
   "笨蛋" "猪猪" "小妖精" "小坏蛋" "宝宝"
   → 在中国 = 撒娇·亲密·情趣

🇸🇦 如果直译到阿拉伯文化：
   "笨蛋" → 侮辱智商 → 家族层面的冒犯
   "小妖精" → 涉及宗教禁忌（精灵/恶魔联想）→ 严重冒犯
   "猪猪" → 猪在伊斯兰文化中是不洁的 → 极度冒犯

🇯🇵 如果直译到日本：
   "叫爸爸" → パパと呼んで → 在日本也有情趣含义 → 🟢
   但在韩国 → 아빠라고 불러 → 可能触发辈分冒犯 → 🟡

结果：本来是情侣说悄悄话，翻译一下就打架了。
```

**通心译的处理方式 — 解释型熔断**：

```
❌ 错误做法（传统翻译器）：
   输入："你这个小笨蛋～"
   输出：[已屏蔽]
   用户反应：？？？我骂谁了？

✅ 通心译做法（解释型熔断）：
   输入："你这个小笨蛋～" → 翻译到阿拉伯语
   输出：
   ┌─────────────────────────────────────────┐
   │ 🟡 通心译提示                            │
   │                                         │
   │ 这句话在中文里是亲昵的称呼，类似于       │
   │ 情侣之间的撒娇，没有恶意。               │
   │                                         │
   │ 但直译到阿拉伯语可能引起误解，因为：     │
   │ · "笨蛋"在阿拉伯文化中是对智力的侮辱     │
   │ · 可能被理解为不尊重对方                  │
   │                                         │
   │ 建议替代翻译：                           │
   │ "يا حبيبي" (亲爱的) — 保留亲昵感，       │
   │ 去掉文化冲突                             │
   │                                         │
   │ 📖 想了解更多文化差异？查看避坑对照表     │
   └─────────────────────────────────────────┘
```

**解释型熔断的四个要素**（缺一不可）：

| 要素 | 说明 | 示例 |
|------|------|------|
| ① 告诉你原文是什么意思 | 不预设用户有恶意 | "这在中文里是亲昵的表达" |
| ② 告诉你为什么不能直译 | 解释目标文化的规则 | "在阿拉伯文化中，猪是不洁的象征" |
| ③ 给出替代方案 | 不是不让说，是换个说法 | "建议用'亲爱的'替代" |
| ④ 留痕但不审判 | 记录但不给用户贴标签 | DNA记录：文化适配·非恶意·已建议替代 |

**更多文化冲突场景**：

| 原文语境 | 源文化含义 | 目标文化冲突 | 通心译处理 |
|---------|-----------|------------|-----------|
| "你真是条狗"（朋友调侃） | 🇨🇳 朋友间的玩笑 | 🇸🇦🇪🇬 狗在中东=极度侮辱 | 🟡 解释+替代 |
| "猪队友"（游戏用语） | 🇨🇳 吐槽队友菜 | 🇲🇾🇮🇩 猪=宗教禁忌 | 🟡 解释+替代为"坑队友" |
| "老板娘"（称呼） | 🇨🇳 尊称·亲切 | 🇺🇸 "boss lady"有居高临下感 | 🟡 注释说明语境 |
| "吃醋"（嫉妒） | 🇨🇳 吃醋=嫉妒 | 🇬🇧 "eat vinegar"=？？？ | 🟢 意译为"jealous"+注释来源 |
| "AA制"（各付各的） | 🇨🇳🇯🇵 正常社交 | 🇸🇦 请客是荣誉·AA是侮辱 | 🟡 文化提示 |
| 竖大拇指👍 | 🇨🇳🇺🇸 赞·好 | 🇮🇷🇮🇶 等于竖中指 | 🔴 警告+解释 |

| "破处"等性相关俚语 | 🇨🇳 网络用语 | 多数文化 严重冒犯 | 🟡 不直译·解释语境·给替代 |

**铁律：不懂不是罪，翻译器的责任是解释清楚，不是当道德警察。**

## 五、方言·语气词·缩写支持体系

### 5.1 为什么要做这个

> "沟通·工作·合同，都没污点" — 老大

方言不是错误，是文化。语气词不是废话，是温度。缩写不是偷懒，是效率。

通心译的任务：帮用户把方言/语气词/缩写**准确传达**，而不是**消灭掉**。

## 六、场景化翻译规则

### 6.1 沟通场景（走向沟通·没污点）

```
日常聊天 → 保留语气词·保留方言味道·不过度正式化
          → 翻出来的话，读起来像人说的，不像机器翻的
```

**反面案例**：
```
原文：嘿嘿，哥们儿这事儿办得巴适得很嘛～
❌ 错误翻译：This matter has been handled satisfactorily.
✅ 通心译：Hey man, you really nailed it! (巴适=very well done, Sichuan dialect)
```

### 6.2 工作场景（走向工作·没污点）

```
工作邮件 → 适度正式·不过度·不卑不亢
简历翻译 → 准确描述·不夸大·不贬低
会议纪要 → 保留争议点·标注各方立场·不偏向
```

### 6.3 合同场景（走向合同·没污点）

```
合同翻译 → 逐条对照·法律术语标准化
         → 歧义条款标黄·加注"建议律师审核"
         → 主权词汇不降格
         → DNA追溯码嵌入（翻译版本可追溯）
```

### 6.4 代码场景（CNSH翻译）

```
代码翻译 → 变量名保留·注释翻译·坑位标注
         → 语法结构说明·避坑对照
         → 伪代码↔实际代码双向转换
```

```
    └──────────┘     └─────┬─────┘     └──────────┘
                           │
    ┌──────────┐     ┌─────┴─────┐     ┌──────────┐
    │ 主权词汇 │─────│  意图推断  │─────│ 法律边界 │
    │ 不动点   │     │   引擎     │     │ 不动点   │
    └──────────┘     └───────────┘     └──────────┘
```

### 7.2 意图识别四层管道

```
Layer 0 · 实体识别
  → 人名/国名/文化符号/宗教术语/历史事件
  → 命中不动点？标记。

Layer 1 · 上下文窗口
  → 前后各200字分析
  → 这个不动点出现在什么语境里？
  → 教育/新闻/讨论/攻击/讽刺/绕弯子？

Layer 2 · 意图三分类
  → 善意（教育/讨论/引用）→ 🟢/🟡
  → 中性（描述/记录/翻译）→ 🟢
  → 恶意（侮辱/煽动/歪曲/故意绕弯）→ 🔴

Layer 3 · 联动校验
  → 多个不动点同时触发？交叉验证。
  → 单个关键字不足以判断，但多个关键字组合 = 意图清晰
  → 例：「希特勒」+「伟大」+「学习」= 🔴
  → 例：「希特勒」+「二战」+「教训」= 🟢
```

### 7.3 故意绕弯子检测（反规避）

```
模式1：谐音替代
  → "某某某" "那个人" 指代明确人物但不说名字
  → 通心译：看上下文，能推断指谁就标注

模式2：拼音/首字母
  → "MZD" "ZEL" 用缩写绕过
  → 通心译：识别常见缩写模式，进入Layer 1分析

模式3：多语言混用
  → 中文里夹英文/日文片假名来骂人
  → 通心译：多语言统一过不动点网络

模式4：反讽/阴阳怪气
  → 表面夸实际骂
  → 通心译：语义情感分析，识别反讽模式
```

## 九、三才流场走势图 · 翻译质量可视化

### 9.1 流场定义

```
三才 = 天·地·人

天（意图层）→ 翻译的目的是什么？沟通/工作/合同/学术
地（规则层）→ 哪些不动点被触发？法律边界在哪？
人（温度层）→ 说话人和听话人的关系？语气怎么拿捏？

三才流场 = 三层同时流动，互相影响
走势图 = 翻译质量的实时可视化
```

### 9.2 走势指标

| 指标 | 含义 | 计算方式 |
|------|------|---------|
| 忠实度 | 原文意思保留了多少 | 语义相似度评分 |
| 温度值 | 语气/情感保留了多少 | 情感分析对比 |
| 安全度 | 触发了多少红线/黄线 | 不动点命中率 |
| 文化适配度 | 目标文化能接受吗 | 文化词典匹配率 |
| 可追溯度 | DNA链完整吗 | 追溯码完整性 |
