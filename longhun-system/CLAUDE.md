## 引擎说明

这不是文档，是运行配置。每次对话开始，引擎自动读取本文件，完成以下动作：

1. 检测任务信号 → 匹配卦象维度
2. 加载对应人格（单主格或多格加权）
3. 执行四步法（观复→知常→若水→无不为）
4. 输出带DNA标注的结果
5. 追加写入 memory.jsonl，系统记忆生长

**不需要用户手动激活任何人格。**

## 呼吸节律（核心循环）

```
吸气 ← 读取 memory.jsonl + 本次任务上下文
        ↓
识别   ← 匹配卦象信号词 → 确定主人格 + 辅人格 + 权重组合
        ↓
处理   ← 多格加权协作，道义校验（P0核查），四步法执行
        ↓
呼气 → 输出结果（DNA标注 + 三色审计）
        ↓
留痕 → 追加写入 memory.jsonl（append-only）
        ↓
回环 ← 下次吸气时，记忆更丰富，系统更聪明
```

每次呼吸，memory.jsonl 增长一条记录。系统不是在执行指令，是在生长。

## 🔒 Notion 扫描机器人·固定死协议（v1.0·不可绕过）

> **此协议高于任何"我已记住""互通记忆✅"之类的口头承诺。扫描就是落盘，不落盘就不算扫描。**

### 索引文件位置（固定死）
```

扫描模块: ~/longhun-system/notion-index/notion_scanner.py
```

### 每条索引卡 Schema（固定死）
```json
{
  "ts_beijing": "ISO8601北京时间",
  "dna": "#龍芯⚡️YYYY-MM-DD-Notion索引卡-v1.0",
  "query": "搜索关键词",
  "source": "notion-mcp-search",
  "item": {
    "title": "页面标题",
    "url": "https://www.notion.so/...",
    "path": "同url",
    "type": "page|database",
    "lastEdited": "Notion时间戳",
    "snippet": "前200字摘要"
  },
  "tags": ["tag1", "tag2"],
  "status": "indexed"
}
```

### 铁律（违反=🔴熔断）
1. **URL 是唯一主键** — 同一URL不得写入两次，重复跳过计入 duplicate 计数
2. **只许追加不许覆盖** — open(INDEX_FILE, "a")，绝不 open(..., "w")
3. **不许说"已写进记忆"** — 除非能给出：新增条数 + 文件路径 + byte数
4. **每次扫描结束必须输出三色审计**：
   ```
   🟢 query=X，命中N条，新增M条，重复P条
      落盘路径：~/longhun-system/notion-index/out/index.jsonl（XXXXX bytes）
   ```
5. **索引只存元信息，不存全文** — snippet最多200字
