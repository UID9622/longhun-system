# 🇨🇳 CNSH编译器语法修复

DNA追溯码：#ZHUGEXIN⚡️2026-01-28-CNSH-SYNTAX-FIX-v1.0

## 🐛 问题描述

生成的C代码出现语法错误：
```c
打印;
"━━━━━━━━━━━━━━━━━━";
```

应该生成：
```c
打印("━━━━━━━━━━━━━━━━━━");
```

## 🔍 根本原因

CNSH语言支持两种函数调用语法：

1. **标准语法**：`打印(「内容」)` ✅ 编译器已支持
2. **语法糖**：`打印「内容」` ✅ 编译器**未**支持

当前编译器只能处理第一种语法，第二种语法会被解析成两个独立的语句。

## 🔧 修复方案

### 1. 添加 `peek()` 方法
在Parser类中添加方法，用于查看下一个token而不消费它：

```javascript
peek() {
  // 查看下一个token，但不消费它
  if (this.pos < this.tokens.length - 1) {
    return this.tokens[this.pos + 1];
  }
  return null;
}
```

### 2. 修改 `parseStatement()` 方法
当遇到标准库函数调用时，检查是否是语法糖形式：

```javascript
if (token.type === 'IDENTIFIER') {
  // 检查是否是标准库函数调用（打印、提示、报错等）
  if (['打印', '提示', '报错', '熔断', '转文本', '拼接', '包含', 
      '当前时间', '记录日志', '生成追溯', '输入文本', '确认'].includes(token.value)) {
    
    // 检查是否是语法糖形式：函数名「参数」（省略括号）
    const nextToken = this.peek();
    if (nextToken && nextToken.type === 'STRING') {
      // 转换为函数调用语法：函数名(参数)
      const funcName = token.value;
      const argValue = nextToken.value;
      this.advance(); // 消费字符串token
      // 返回一个函数调用节点
      return new ASTNode('ExpressionStatement', {
        expression: new ASTNode('FunctionCall', {
          name: funcName,
          args: [new ASTNode('StringLiteral', { value: argValue })]
        })
      });
    }
    return this.parseExpressionStatement();
  }
}
```
