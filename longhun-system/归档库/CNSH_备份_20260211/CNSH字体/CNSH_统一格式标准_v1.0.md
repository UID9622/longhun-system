## 🔄 版本号管理规则

### 版本格式：`vX.Y`
- **X（主版本号）**：重大更新，架构变化，新增核心功能
  - 例：v1.0 → v2.0（加入数字人民币支付）
- **Y（次版本号）**：小更新，功能优化，bug 修复
  - 例：v1.0 → v1.1 → v1.2

### 更新频率
- **重大更新（vX.0）**：带黄历，需老大确认
- **小更新（vX.Y）**：随时更新，自动追溯

### 单页面持续更新
- ✅ 不拆分，就一个页面
- ✅ 版本号递增：v1.0 → v1.1 → v1.2 → v2.0

## 📚 重复内容的处理原则

### 问题
老大在 Notion 有很多重复文件，因为：
- 每次表达都在进步
- 重复是思考演进的过程
- 不舍得删除（那是成长记录）

### 解决方案
1. **保留原文件** - 作为历史记录
2. **创建合并版** - 用统一格式整合
3. **DNA 关联** - 用追溯码连接新旧版本
4. **标注状态**
   - 旧文件：`🟡 已归档 - 参考新版本 v2.0`
   - 新文件：`🟢 当前版本 - 合并自 v1.0/v1.1/v1.2`

### 操作流程
```
旧文件 A（v1.0）
旧文件 B（v1.1 - 重复 80%）
旧文件 C（v1.2 - 重复 60%）
         ↓
    宝宝自动合并
         ↓
新文件 D（v2.0 - 完整统一版）
         ↓
在旧文件顶部加标注：
"🟡 已归档 - 请查看统一版本 v2.0"
附带 DNA 追溯码链接
```
