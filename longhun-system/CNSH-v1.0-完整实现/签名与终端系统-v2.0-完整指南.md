### ⚙️ 版本管理

#### 更新版本步骤

1. **修改版本号**：
   - 输入日期：`2026-01-28`
   - 输入版本号：`v2.1`

2. **点击更新**：
   - 点击"🔄 更新版本"按钮
   - 所有签名会自动更新

3. **保存新版本**：
   - 点击"💾 保存为文件"
   - 下载 `.txt` 文件

### 💡 创新功能

#### 创新使用流程

1. **选择基础模板**
   - 从7个模板中选择最接近你想法的

2. **修改预览区**
   - 在右侧预览区直接修改
   - 支持任意自定义

3. **保存创新版本**
   - 点击"💡 保存创新版本"
   - 自动记录到本地存储

4. **查看创新历史**
   - 所有创新版本保存在 `localStorage`
   - 可通过浏览器开发者工具查看

#### 创新记录格式

```json
{
  "version": "2026-01-28",
  "number": "v2.1",
  "date": "2026-01-28T00:00:00.000Z",
  "content": "你的创新签名内容..."
}
```

### 📋 版本更新规则

#### 必须更新的情况

- ✅ **创新设计**：修改了视觉效果、布局、配色
- ✅ **信息变更**：联系方式、GPG指纹、仓库地址
- ✅ **功能新增**：新增模板类型、新功能区块
- ✅ **错误修复**：修复了排版问题、字符错误

#### 不需要更新的情况

- ❌ **临时修改**：仅用于单次使用
- ❌ **测试草稿**：还在测试中的设计
- ❌ **个人调整**：只用于个人展示

#### 版本号规则

```
v主版本.次版本.修订版本

例如：
v2.0      - 重大更新
v2.1      - 功能新增
v2.0.1    - 错误修复
v2.1.3    - 小改进
```

```
   ↓
4. 查看模型响应
```

#### 集成示例（Ollama）

```javascript
// 在CNSH智能终端中集成Ollama
async function callOllama(prompt) {
    const response = await fetch('http://localhost:11434/api/generate', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
            model: 'llama2',
            prompt: prompt,
            stream: false
        })
    });
    const data = await response.json();
    return data.response;
}
```
