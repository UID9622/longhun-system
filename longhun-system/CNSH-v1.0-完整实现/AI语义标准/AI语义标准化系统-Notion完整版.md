## 2️⃣ 预处理层（Pre-Processing）

### 2.1 语言降噪算法
**处理规则：**
```yaml
口语填充词：
  - 剥离：嗯、啊、那个、就是
  - 保留标记：可能影响情绪判断的词

重复表达：
  - 压缩连续重复（3次以上）
  - 保留首次出现

非语义情绪词：
  - 标记但不删除：妈的、靠、卧槽
  - 用于情绪分析
```

### 2.2 语法结构规整器
**处理规则：**
```yaml
断句重构：
  - 按标点符号拆分
  - 超长句自动分段

隐含主语补全：
  - "想搞一下" → "[我]想搞一下"
  - "不太行" → "[这个]不太行"

指代消解：
  - 他 → 查找最近提及的人
  - 这个 → 查找最近提及的事物
  - 失败时标记"指代不明"
```

### 2.3 多义词歧义缓冲区
**策略：**
- 不立即定义含义
- 保留多个候选解释
- 延迟到意图识别层决策

**示例：**
```
输入：搞一下
候选含义：
  1. 执行某操作（工作语境）
  2. 修改某内容（技术语境）
  3. 处理某问题（决策语境）
延迟到意图层选择
```

## 4️⃣ 意图识别层（Intent Layer）

### 4.1 主意图分类器
**字段：**
| 字段名 | 类型 | 选项 |
|--------|------|------|
| Main Intent | Select | 请求信息/请求决策/请求执行/表达观点/情绪宣泄/试探暗示 |
| Intent Confidence | Number | 0-1 |

**分类规则：**
```yaml
请求信息：
  - 疑问词：什么、怎么、为什么
  - 示例：这个怎么弄

请求决策：
  - 判断词：应该、要不要、可不可以
  - 示例：我们应该用哪个

请求执行：
  - 动作词：做、搞、执行、运行
  - 示例：帮我搞一下

表达观点：
  - 陈述句+评价词
  - 示例：我觉得这个不错

情绪宣泄：
  - 高情绪+无明确请求
  - 示例：太tm难了

试探暗示：
  - 委婉表达+未明说
  - 示例：不知道你能不能...
```

### 4.2 隐含意图挖掘
**策略：**
- 未显式说出的目标
- 行为期望（回应/行动/确认）

**示例：**
```
显式：帮我看一下这个代码
隐含：期望找出bug或优化建议
```

### 4.3 意图置信度评分
**规则：**
```yaml
高置信（>0.7）：
  - 直接进入执行
  
中置信（0.4-0.7）：
  - 复述确认后执行
  
低置信（<0.4）：
  - 进入澄清模式
  - 不执行任何操作
```

## 7️⃣ 真实度与可信度层（Reality Layer）

### 7.1 真实度增强器
**处理规则：**
```yaml
去除夸张：
  - "绝对" → "可能"
  - "一定" → "应该"
  - "永远" → "通常"

标记假设条件：
  - 显式添加"如果...则..."
  
明确不确定性：
  - "可能"、"大概"、"估计" → 保留
  - 不增加虚假确定性
```

### 7.2 事实与推断分离器
**输出格式：**
```yaml
已知事实：
  - 直接陈述
  - 高置信度
  
推断内容：
  - 标记为"根据...推断"
  - 中置信度
  
猜测内容：
  - 显式标注"猜测"
  - 低置信度
```

**示例：**
```
输入：他应该是不同意这个方案
输出：
  - 已知事实：方案被提及
  - 推断内容：根据语气推断对方可能不同意
  - 猜测成分：对方态度未明确表达
```

## 1️⃣2️⃣ Fail-Safe守则与异常处理

### 7条不可违背守则
```yaml
守则1｜不确定即降级:
  - 触发条件：置信度 < 阈值
  - 动作：降级为复述/澄清结构

守则2｜不理解不执行:
  - 触发条件：意图置信度 < 0.7
  - 动作：禁止执行型输出

守则3｜不猜隐含前提:
  - 触发条件：未显式给出的条件
  - 动作：一律不补全

守则4｜多冲突取单一:
  - 触发条件：多意图/多主语
  - 动作：仅保留最高置信路径

守则5｜情绪不主导:
  - 触发条件：高情绪检测
  - 动作：情绪仅影响节奏，不影响语义

守则6｜未知不映射:
  - 触发条件：未知术语
  - 动作：原样保留 + Unknown标记

守则7｜稳定优先于完整:
  - 触发条件：系统不稳定
  - 动作：系统稳定性 > 信息完整性
```

### 异常类型与处理矩阵
| 异常类型 | 触发条件 | 风险 | 兜底策略 | 错误码 |
|----------|----------|------|----------|--------|
| 输入过短 | <3词 | 意图无法判断 | 标记低语义密度 | E001 |
| 输入过长 | >2000 tokens | 解析成本过高 | 启动压缩+主干提取 | E002 |
| 多语言混杂 | 语言混用 | 语义错配 | 选择占比最高语言 | E003 |
| 噪声过载 | 噪声>语义 | 输出空洞 | 保留原句+低可解析标记 | E101 |
| 断句失败 | 结构混乱 | 逻辑错乱 | 回退原句顺序 | E102 |
| 指代无法消解 | 指代不明 | 指错对象 | 显式标记"指代不明" | E201 |
| 多主语冲突 | 主语冲突 | 意图漂移 | 强制单主语模式 | E202 |
| 情绪极高 | >0.8 | 冲动回应 | 启用降速+中性语义 | E301 |
| 情绪类型不明 | 无法识别 | 错误安抚 | 不启用情绪话术 | E302 |

### 常见问题速查表
| 症状 | 可能原因 | 诊断方法 | 解决方案 |

| Fail-Safe频繁触发 | 输入质量低或阈值过严 | 查看触发日志 | 调整阈值或优化输入 |
| 情绪检测不准 | 情绪词库不完整 | 查看 Emotion Type 误判案例 | 补充情绪词库 |
| 响应时间过长 | Compute Mode = Deep | 查看 Processing Time | 切换到 Stable 模式 |
| 术语映射失效 | 优先级设置错误 | 查看 Terminology Map 优先级 | 调整优先级 |
